## 🟥 C-1 · 历史概述全站太简洁（确认 · 所有城市）

### 现象
罗马、北京、伊斯坦布尔、马丘比丘…… 每个城市概述都是 3-4 行骨架。
示例（罗马）：
> "罗马横跨三千年，从传说中罗慕路斯建城到罗马帝国、教皇国直至现代意大利首都。这座城市曾是已知世界的政治、宗教与文化中心，其遗迹至今仍与现代都市生活并存共生。"
然后——就没了。直接跳到时间轴。

### 用户视角诊断
概述是**钩子**，钩住以后**没有故事接住**。我打开这个 App 想读"这座城为什么是它现在的样子"——我想读的是**散文**，不是年表。
- **时间轴回答 "what / when"**
- **概述展开应该回答 "why"**
两个维度不能互相替代。现在缺"why"。

### 方案（给 PM 写 PRD）

**组件规格**：
- 概述区域保留现有 4 行短钩子（不变）
- 末尾右侧加一个**极克制的金色"继续阅读 →"**链接
  - 字号 13px / 细描边 / 0.7 透明度
  - 点击展开一段 **300-500 字散文**
- 展开内容样式：16px 衬线中文 / 1.8 行距 / AAA 对比度（和现有正文一致）
- 展开/收起带 0.4s ease 动效

**内容规格（需要 PM 或内容团队补写）**：
每个城市一段 300-500 字，**用"为什么"串起来**：起源 → 转折 → 衰落 → 复兴 → 今天。
罗马示范（我写给 PM 参考）：
> 七丘之城起于一个传说——两个被母狼哺乳的弃婴在台伯河边立起了城。接下来的一千年，罗马从一个拉丁部落联盟，变成整个地中海的主人。但真正塑造现代罗马的不是奥古斯都，而是公元 4 世纪那次改变——当君士坦丁把基督教从被迫害的边缘信仰，变成帝国的官方宗教时，这座城就同时成为了"罗马"与"天主教罗马"。从那以后的 1600 年里，帝国消失了，教皇接管了，文艺复兴爆发了…… 但罗马始终有个怪脾气：它从不把过去拆掉重建，它让每一层过去都叠在上一层上。走在今天的罗马街头，你脚下 10 米是古罗马广场，头顶是巴洛克穹顶，旁边是文艺复兴别墅，转角是墨索里尼修的理性主义火车站——这是一座**没有把自己杀死过一次**的城市。

### 类比锚点
- **The School of Life 城市指南** — 每座城一段"性格画像"
- **Monocle Travel Guide 开篇页** — "让你上瘾的是什么"
- **Cereal 杂志城市专题** — 散文+照片的呼吸感

### 优先级
**P1** · 这是这个 App 真正的护城河。内容量大（10+ 城市 × 500 字 = 5000 字），建议 PM 分两批写。

## 🟥 C-2 · "世界其他地方"板块错位（景点 → 历史概述）

### 现象
现在每个景点卡展开后，底部有一块"🌍 世界其他地方"—— 斗兽场下面写"公元 72 年，东汉洛阳在修宫殿"。**用户（我）在景点卡读到这段会出戏。**

### 用户视角诊断

**为什么放在景点卡里是错的**：
我正在看斗兽场——血腥的角斗士、椭圆竞技场、今天怎么买票——这是**一个具体文物的沉浸镜头**。这时候突然弹出"公元 72 年，东汉洛阳在做什么"——**相当于电影里突然切到纪录片画外音**。尺度不对：**定点 vs 平面**，不兼容。

**为什么放在历史概述就完全对路**：
- 历史概述本来就是宏观时间线 + 文明跨度
- 同时期世界对照是**时间的切片**——天然属于宏观视角
- 读者读到"罗马帝国鼎盛（公元 100 年前后）"时，横向看到"东方：东汉班超经营西域 / 美洲：玛雅蒂卡尔崛起 / 南亚：贵霜王朝丝路盛期"——**这才是纪录片感**
- 这是 *Sapiens* / *枪炮病菌与钢铁* / BBC《文明》的核心叙事方法：**文明同步性**

### 方案（给 PM 写 PRD）

**重构动作**：
1. **撤掉**景点卡展开时的"世界其他地方"整块（让景点卡更聚焦斗兽场本身 → 门票 / Tips 去填这个空间）
2. **迁移**到时间轴——每个重大节点旁边横向延伸细金条，展示 2-3 个同时期其他文明在做什么

**时间轴节点样式升级**：
```
  公元 80 年 · 斗兽场落成
  ─────────────●────────────
                │
  同时期世界    │  ← 细金条 / 12px / 半透明
  · 东汉：班超出使西域
  · 玛雅：蒂卡尔立起第一批石柱
  · 贵霜：丝路东西贯通鼎盛期
```

或者更克制的方案：时间轴节点旁加一个小的 `⌘ 同时期世界 →` 按钮，点击才展开同时期对照——默认折叠，不喧宾夺主。

### 类比锚点
- **BBC《文明》纪录片** — 每集的核心结构就是"同一时刻，世界各地"
- **大英博物馆"世界史 100 件"播客** — 按时间切片讲，不按地理
- **Sapiens 章节结构** — 以时间为横轴，文明为纵轴

### 优先级
**P1** · 这是这个 App 独家的内容优势，目前放错位置。建议和 C-1（概述展开）一起重构。

## 🟥 C-3 · 门票购买 & 预约 tips 完全缺失

### 现象
斗兽场门票栏现状：
> 🎫 €18（含古罗马广场联票）· 09:00—19:00 · 建议网上预约跳过排队

**三行，等于 0 信息深度**：
- 没有官方 URL
- 没有授权平台推荐（Klook/Tiqets）
- 没有预约提前窗口
- 没有淡旺季区分
- 没有时段建议

### 用户视角诊断

打开一个旅行 App，**90% 的真实意图是"我要去 / 可能会去"**，不是"被科普"。没有购票路径 = 这个 App 在"决策 → 行动"链条上**断掉**。用户路径会变成：
1. 翻到斗兽场页面 → 看着美 → 心动
2. **退出 App** → 打开浏览器搜"斗兽场门票怎么买" → 到马蜂窝/Tripadvisor
3. 我们**流失了这个用户**

### 方案（给 PM 写 PRD）

**每个付费景点最小完整信息 4 件套**：

| 字段 | 数据类型 | 示例（斗兽场） | 数据来源（Dev research） |
|---|---|---|---|
| `ticket.price` | string | "€18（联票含古罗马广场+帕拉蒂尼山，48h 内有效）" | 官网 |
| `ticket.channels` | array of {name, url, note} | [{官方 colosseo.it, 最便宜但系统不稳}, {Klook, 中文可取消}, {Tiqets, 快速预约}] | 实际测试 |
| `ticket.bookingWindow` | object | {peak: "旺季 2-4 周", shoulder: "春秋 3-7 天", offpeak: "淡季 1-3 天"} | 马蜂窝评论 + Reddit |
| `ticket.timingTip` | string | "开门前 8:30 到，或落日前 1 小时最美" | 小红书博主实战帖 |

**UI 样式建议**：
```
┌─ 票价 ────────────────────────
│  €18  联票含古罗马广场+帕拉蒂尼山，48h
├─ 购票渠道 ────────────────────
│  ↗ 官方 colosseo.it   最便宜 · 系统偶尔崩
│  ↗ Klook              中文 · 可取消
│  ↗ Tiqets             快速预约
├─ 预约提前 ────────────────────
│  旺季 7-8月  2-4 周
│  春秋肩季    3-7 天
│  淡季 12-2月 1-3 天
├─ 时段建议 ────────────────────
│  最佳：开门前 8:30 · 或落日前 16:30 后
│  避开：11-14 团客高峰
└──────────────────────────────
```

用户已经给出明确 research 源：**马蜂窝 / Tripadvisor / Reddit r/travel / 小红书**。PM 可以按这个列表指导内容团队/Dev 收集。

### 类比锚点
- **Klook 详情页** — 每景点标配 3 档购票通道
- **Tripadvisor 票务卡** — "Book in advance" 具体到天数
- **Airbnb Experiences** — "This experience usually sells out 2 weeks ahead"

### 优先级
**P0** · 这是 App 从"画册"升格到"工具"的**唯一关键**。没这个，Monocle 级美学只是纸上谈兵。

## 🟧 C-4 · 每个景点 Tips 太单薄（只有一条 · 全是常识）

### 现象
斗兽场现有 Tips 只有一条：
> 💡 强烈建议网上预约跳过排队

这是**常识**，不是 Tip。一条常识 tip = 不如没有，因为它让用户以为"这个 App 的深度也就到这里了"。

### 用户视角诊断

**Tips 的真正价值**：只有**去过的人才知道**的那种内行细节。
- 常识 tip（"网上预约"）= 对老手毫无价值
- 内行 tip = 真正的情感锚点，用户会**截图发朋友圈、推给朋友**

### 方案（给 PM 写 PRD）

**每个景点 4-6 条分类 Tips**。分类和图标：

| 类别 | SVG 图标 | 典型内容（斗兽场举例） |
|---|---|---|
| 🎫 票务技巧 | ticket-dashed | "门票包含古罗马广场，48h 内有效，分两天玩透省票钱" |
| ⏰ 时段内行 | clock-thin | "开门前 8:30 到，或落日前 1 小时最美" |
| 📸 最佳机位 | camera-line | "从康斯坦丁凯旋门方向拍西立面，夕阳打光" |
| 🚶 动线串联 | footprint-line | "出斗兽场 → 真理之口（B 线一站）→ 特莱维喷泉" |
| 👔 着装/规矩 | shirt-line | "进梵蒂冈需遮肩遮膝，斗兽场无此要求" |
| ❄️ 季节差异 | snowflake-line | "12-2 月无排队、票价不变，下午 4 点天黑" |
| 🕳 隐藏彩蛋 | key-line | "地下层 Hypogeum 需额外 €9，能看角斗士通道" |

**数据模型（建议 PRD 里规范）**：
```js
landmark.tips = [
  { category: 'ticket', text: '门票包含古罗马广场...' },
  { category: 'timing', text: '开门前 8:30 到...' },
  { category: 'photo',  text: '从康斯坦丁凯旋门...' },
  { category: 'route',  text: '出斗兽场 → 真理之口...' },
  ...
]
```

**UI 样式**：
- 米色半透明 list（像杂志的 "Editor's Note"）
- 每条一行 · ≤30 字 · 分类 SVG 图标在最左
- 整体图标系统 stroke-width 1.5（和现有一致）

**重要内容约束**（写进 PRD）：
- ❌ **禁止常识 tip**（如"建议网上预约"、"请穿舒适鞋"）
- ✅ **要内行细节**：数字、时间、具体方向、彩蛋

### 类比锚点
- **Cereal 杂志 "Insider Notes"** — 每城 8-10 条本地人视角
- **小红书博主张阿瑶的旅行帖** — 每条都是"大家都不知道但真有用"
- **反面教材：Lonely Planet** — tips 太多太常识，沦为列表

### 优先级
**P0** · 性价比最高的内容升级。每景点只需 1-2 小时 research。

## 一点尺度提醒

上面 4 条**全部是产品结构动刀**，不是样式微调。涉及：
- 数据模型扩展（`ticket.*`、`tips[].category`、`sameTime[]`、`overviewLong`）
- 布局组件重做（景点卡、时间轴节点、概述展开）
- **内容补写量巨大**（每城市 500 字 × 10+ 城市，每景点 4-6 条 tips × 50+ 景点）

所以**不要四件事同时做**会乱。建议：
1. **C-3 + C-4 先做**——只改景点卡 + 补 tips/门票 research，一周内见效
2. **用户感觉"内容密度涨了"后**，再启动 C-1 + C-2 的叙事重构
