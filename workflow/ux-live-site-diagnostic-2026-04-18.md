# 🚨🚨🚨 P00000 · CEO 亲自点名 · 好几个版本未修的死结

**问题**：景点介绍页的"是什么"展示塌方式崩溃——好几个版本没修，CEO 已明确表示"受不了了"。

**CEO 原话引用**：
> "就是景点的介绍首页展示，说了很多次，**文字太多、文字太大、字体奇怪、可读性很差，没有分级分段**。"
> "我受不了了！P00000级解决！！"

**现场证据**（2026-04-18 CEO 截图 3 张）：
- 华纳比丘 · 印加时期 · "是什么"段约 350 字堆成一坨
- 拴日石（Intihuatana）· 印加 · 约 1450 年 · "是什么"段约 400 字堆成一坨
- 金阁寺（鹿苑寺）· 室町 · 1397 年 · 相对好（有"为什么独特""跨文明"分段），但仍单段过长

**问题拆解**（对应我 6 个目标的多条镜头）：

| 子问题 | 对应镜头 | 具体观察 |
|---|---|---|
| A. 单段过长 | 段落长度红线（已退役，需恢复）+ 图文交替呼吸 | 单段 300-400 字无分段无图片打断，移动端滑 3-4 屏才到底 |
| B. 字体奇怪 | 字体安静检验 | 衬线字体（看起来像系统默认 serif）在 CJK 环境下读着**喧哗**，字号偏大放大了不适 |
| C. 字号偏大 | 字重分层 > 字号跳级 | 正文字号目测 18-20px（移动端理想 15-16px），和标题区分度不足 |
| D. CJK 行距偏小 | CJK 行距加码 | 目测行距 ≈1.5 或更低，CJK 需 1.7-1.8 才不挤 |
| E. 没有分级 | 渐进披露层级 + 密度递减方向 | "是什么"框里混了建造年代、地理位置、历史故事、关键数字、跨文明对照——全挤在同一段 |
| F. 配色老气 | 颜色是路标不是装饰 | 深绿底 + 黄色引导色 + 暗淡正文 = 学报/政府文件感，不是 Monocle 编辑感 |
| G. 金阁寺做对了的部分没推广 | 渐进披露 | "为什么独特""跨文明"小标题分段是**正确方向**，但只在金阁寺实现，其他景点没跟进 |

**UX 展示要求（下次上线必拍）**：
1. 3 张截图取证（华纳比丘、拴日石、金阁寺各一张完整"是什么"段）
2. 一张**正面对照**：金阁寺的分段版本 vs 拴日石的无分段版本
3. 开发者工具测一次 `line-height` 和 `font-size` 数值（打开 Inspect 截图）
4. 一段慢速滚动录屏，展示"一屏读不完一段"的真实感受

**期望修复方向**（给 Dev-H5 / PM 参考，不是强制）：
- **把金阁寺的"为什么独特 / 跨文明 / 现场细节 / 古今连接"四段式结构推广到所有景点**（数据结构 `whyVisit` 或类似字段已支持，缺的是渲染层）
- **正文字号降到 15-16px**，行距提到 1.75
- **字体换成更克制的中文字体栈**（建议：`-apple-system, "PingFang SC", "Noto Sans SC", system-ui, sans-serif`，不用衬线 serif）
- **单段严格 ≤150 字**，超过必须拆段或插图

**升级路径建议**（research-ux 无权限操作，请 PMO 执行）：
- 直接进 `workflow/backlog.md` 作为 **P00000 · CEO 亲自点名**
- Dev-H5 下一轮最优先处理
- 修完 QA 必须把 3 张页面的前后对比截图附在 QA 报告里

# P0 · 已知问题（已在多轮 UX/QA 反馈里出现，未彻底解决）

## 目标 1 · 流畅度

### 1.1 打字机动效 + Hero 图加载不同步
- **已知**：v2.7 反馈"打字机文字先蹦出，图加载拖拉"
- **展示**：首次进城市页全程录屏 + 0.25× 回放
- **期望**：打字机应等到 Hero 图 onload 后再触发，或者两者节奏重新编排

## 目标 4 · 文字易读（P00000 之外的独立问题）

### 4.1 whyVisit 斜体 150-200 字无折叠
- **已知**：v2.6 反馈
- **展示**：详情页截图
- **和 P00000 的关系**：属于同一族问题，但这条是"斜体"独有——斜体 + 长段 + 无折叠是 CJK 排版三连击

### 4.3 relatedLiterature 三段论文墙
- **已知**：v2.7 反馈
- **展示**：详情页底部截图
- **期望**：拆成卡片式 3 张，每张 ≤2 行摘要 + 一张封面

## 目标 5 · 产品美观

### 5.1 "/现在："斜线破坏排版
- **已知**：v2.8 反馈
- **展示**：元素截图
- **期望**：改为独立一行的小标题而非行内斜线

### 5.2 MEANWHILE emoji 三连
- **已知**：v2.8 反馈 "从 Monocle 级滑到 Slack"
- **展示**：元素截图
- **期望**：换成 SVG monoline icon，或纯文字 label

## 目标 6 · 图片大陆可用

### 6.1 京都、吴哥窟 Hero 图黑屏（ORB 拦截）
- **已知**：多轮反馈
- **展示**：截图 + 控制台 Network tab 报错
- **期望**：所有图片资源迁国内 CDN，墙外源全部淘汰或加代理页

### 6.2 5 张景点图 ORB 拦截
- **已知**
- **展示**：网络面板 status 截图（红色 failed）
- **期望**：同 6.1

### 6.3 图片加载无 aspect-ratio 占位（慢网络下内容跳动）
- **已知/怀疑**
- **展示**：Throttle → Slow 3G → 录屏
- **期望**：每张图 CSS 层设 `aspect-ratio`，避免 CLS

# P1 · 怀疑但需 UX 证实

## 目标 1 · 流畅度

### 1.2 缓动曲线全站是否统一
- 列表→详情→返回三次触发，对比动画尾部"刹车感"
- 期望：定义一条 `cubic-bezier(.2,1,.2,1)` 全站复用

### 1.3 回退保 scroll
- 滚 5 屏后点详情再返回，是否回原位？
- 期望：`scrollRestoration = 'manual'` + sessionStorage 存位

### 1.4 共享元素过渡
- 列表卡片图点开是"长大"还是新页面冒出？

## 目标 2 · 前端展示架构

### 2.1 首页入口轴数
- 是否只有"先选城市"1 条？还是 ≥3 轴（时期/主题/大洲/情绪）并列？
- 展示：首页截图 + 数路径

### 2.2 详情页一屏铺满
- 所有信息（whyVisit+ticket+timeline+figure+literature）是否默认全展开？
- 期望：渐进披露——默认只看 whyVisit，其他点击展开

## 目标 3 · 颜色配色

### 3.1 亮色出现次数
- 内容区亮色（非黑灰白）≤1 处（功能色）还是 ≥3 处（装饰色泛滥）？
- 展示：首页 + 详情页圈出所有亮色

### 3.3 配色是否系统化
- 是 Stripe 式 5 级对比规则，还是"看起来好看"随手选的？

## 目标 4 · 文字易读

### 4.4 层级是字重还是字号跳
- 各层级并列截图，数字重档位

### 4.5 移动端图片全宽 bleed
- 图片突破内容 padding 延伸到屏幕边缘，还是被框住？

## 目标 5 · 产品美观

### 5.3 卡片 overlay 密度
- 城市卡片上叠加元素（徽章/emoji/角标）≥3 个？
- 展示：首页卡片截图

## 目标 6 · 图片

### 6.4 第一张图选择
- 城市 hero 是"地标全景明信片"还是"局部切片"？
- 展示：所有城市 hero 截图并排

### 6.5 图片渐进加载美学
- 是否有 blur-up / dominant color / LQIP？
- 展示：Slow 3G 录屏

# UX 执行建议

## 流程

1. 开工前读这份诊断 + `workflow/ux-lenses.md`（仍然必读，镜头库是锚点）
2. 上线后按**模式 A（EXPERIENCE）** 照常自由逛（保留 3 秒直觉）

4. 写反馈文件 `用户反馈-2026-04-XX-HHMM.md` 时：
   - 前半部分是**自由体验**（保留直觉）
   - 后半部分是**诊断回收**（按本清单编号回应）

## 特别纪律

- **P00000 必须最先拍 + 最详细**
