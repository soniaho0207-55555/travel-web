## 你（CEO）只需要知道的 3 件事

### 1. 代码在哪看
- **想看线上真实的代码** → `main` 分支
- **想看 Dev 正在搞什么** → `dev` 分支
- 两者差异 = 还没上线的功能

### 2. 一个功能从想法到上线，走这条路
```
想法 → backlog.md  → Dev 在 dev 写 → QA 测 dev → PMO 合到 main → 上线
                                                    ↑
                                            这一步之前，用户看不到
```

### 3. 什么时候分支会"多起来"
只有 Dev 做**大改动**（比如重构），才会临时开 `feature/xxx` 或 `fix/xxx` 分支，做完合回 dev，然后**立刻删掉**。不然就是僵尸。

**规矩：合并即删除。** 今天清的那两个就是没遵守这条规矩留下的。

## 常见问题

**Q: 我不小心在 main 改了代码怎么办？**
A: 告诉 PMO，不要 push。PMO 会把你的改动搬到 dev 再回退 main。

**Q: dev 上的东西会不会影响线上？**
A: 不会。dev 是草稿本，只有 PMO 主动合并才会上线。

**Q: 为啥不直接在 main 上开发？**
A: 防止写一半的代码被用户看到。dev 是"保护垫"。

**Q: 这些分支占我磁盘吗？**
A: 几乎不占。一个分支就是一个指针，不是整份代码拷贝。放心留着。
