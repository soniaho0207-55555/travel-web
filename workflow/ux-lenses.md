# UX 眼力镜头

UX 每轮 pipeline 开工前**必读**的"小本本"。研究成果浓缩在这里，让 UX 的眼睛越来越锐利。

---

## 规则（写入前必看）

- 每条 **≤1 句话**，必须是"下次看到 X 就该亮红灯"的**可观察行为**
  - ✅ 锐：`城市页第一屏：是"标题+图"还是"打动人的句子+图"？`
  - ❌ 糊：`要有呼吸感` / `图文要匹配` / `排版要优雅`
- 每条末尾加 `→ 来源：research/<文件名>.md`
- 每段上限 **15 条**，满了要淘汰最旧或最少被激活的（淘汰记到文末"历史淘汰"）
- **只有 `/research-ux` session 能写这个文件**；`/be-ux`、pipeline 的 ux-tester subagent 只读

## 🔵 长期镜头（美学 / 流畅度 / 惊喜，研究沉淀）

- 回退保 scroll：滚 5 屏后点进详情，按回退——回到原位置 or 跳顶？跳顶 = "你刚才看的东西不重要"信号 → 来源：research/流畅度-Linear-2026-04-16.md
- 渐进披露层级：卡片默认只展示 1 级信息（名字+一句话+图），详情打开才见 2 级（描述/票价），主动展开才见 3 级（地图/交通）？一屏全铺 = 电话簿 → 来源：research/美学-Things-3-2026-04-17.md
- 字重分层 > 字号跳级：层级靠 weight（semibold/regular/light）+ 颜色（黑/灰）两轴区分，还是每级跳 4px 字号？后者 = 层级靠喊叫 → 来源：research/美学-Things-3-2026-04-17.md
- 图片占位美学档次：加载空档看到什么？白屏/菊花=最差；灰 skeleton=中；主色填充=中上；LQIP 模糊低清图=上（Medium 级）；CSS-only LQIP=顶。问自己"这 2 秒我想关 tab 吗" → 来源：research/图片-渐进加载美学对比-2026-04-19.md
- 动画属性白名单：过渡动画只动 transform 和 opacity？还是也动 height/width/margin？后者 = GPU 掉队、卡顿破绽 → 来源：research/惊喜-Stripe-官网-2026-04-17.md
- 图片域名是否国内：Network 面板看图片 host，是 `*.myqcloud.com` / `*.aliyuncs.com` / `*.volccdn.com` / `img.brand.com` 还是 `googleusercontent.com` / `unsplash.com`？后者 = 墙外源，大陆可用性赌运气 → 来源：research/图片-国内CDN对比-2026-04-18.md
- Mobile-first responsive Grid fallback 完整性：Grid 容器（≥2 列）在 ≤768px 必须**完整重置 5 项**（①grid-template-columns: 1fr ②align-items 不继承 desktop center ③aSide/bSide 文字方向统一 left-align ④装饰元素重新计算 hinge 横线方向/长度 ⑤中央 link/hinge 文字桌面连接句"一个 X·一个 Y"在窄屏必须隐藏/重写/改装饰条）+ 同卡字体家族 ≤2 套 / 字号档位 ≤3 级（A 全局约束）；只换 columns 不换其他 = "被压扁的桌面"非真 mobile-first → 来源：research/taste-CEO-对照卡-镜头诊断-2026-04-24.md
- 图片格式兜底：`<picture>` + srcset 有没有 AVIF / WebP / JPEG 三级降级？只有 JPEG = 浪费 40-60% 流量 + 加载慢；只有 WebP 无 fallback = 旧设备看不到 → 来源：research/图片-国内CDN对比-2026-04-18.md
- 缓动曲线人格：全站动效用同一条自定义 cubic-bezier 还是每处随手写 ease / linear？后者 = 没有动效人格，每个动画像出自不同团队 → 来源：research/惊喜-Stripe-官网-2026-04-17.md
- 密度递减方向：从导航区到内容阅读区，信息密度是递减的（导航紧凑/阅读松弛）还是全页统一密度？统一密度 = 一刀切，没有功能区分 → 来源：research/美学-Bear-2026-04-17.md
- 字体安静检验：读完一屏内容后你记得字体长什么样吗？记得 = 字体太抢戏；忘了 = 字体在正确地支撑内容（Bear Sans 的设计目标：sit silently） → 来源：research/美学-Bear-2026-04-17.md
- 图片加载韧性：`<img>` 同时做到 `aspect-ratio` 撑高度（防 CLS）+ `onerror` 兜底占位图（防失败空白）？两项都有 = 韧性到位；缺一 = 加载失败时 layout 崩 / 布局抖动 → 来源：research/产品动线-H5装native-2026-04-16.md + research/图片-墙外源ORB拦截机制-2026-04-18.md
- Tab 内部三层密度递减：进 tab 第一屏是索引层（标题+定位+数量，高密度）→ 中段阅读层（body+细节图，低密度）→ 底行动层（时段/CTA，中密度）？一进 tab 就陷长文 = 分层缺失 → 来源：research/taste-αβ-结构+图片模式-2026-04-22.md
- 深度层 single-focus 原则：深度层（可折叠 section）是"1 个主题深到末端"（Cereal sidebar 工艺拆解 / LP secondary box 历史上的一天 / GAC deep dive 长文）还是"横向扫 3-5 条一行"（worldEvents + meanwhile + relatedFigure 列表）？后者是外围层伪装深度；真深度必须 single-focus + 下沉末端 → 来源：research/taste-深度层+参观时段-2026-04-22.md
- 字体台阶克制：同模块字号台阶 ≤3 级 · 字体家族 ≤2 套（含 blockquote 的 cite 默认衬线 / display serif 的 CJK fallback Noto Serif SC 也算 1 套）· **页面级**单页字号档位 ≤3 级（爆 8 级是物理弃读）；深度层出现 h3/h4/paragraph/pullquote/cite 4-5 级独立样式 = 必然"字体不和谐"；西文 high-contrast display serif（Playfair Display / Cormorant / EB Garamond）+ CJK fallback 中文落衬线、与同卡 sans bullets 不一致 = 字体台阶违反 → 来源：research/taste-v7-线上验证-2026-04-23.md + research/taste-CEO-对照卡-镜头诊断-2026-04-24.md

## 🟢 旅游垂直镜头

- 城市页第一屏开场：纯名词标题（"巴黎"+图）= 旅游目录感；带视角/感官/时刻的一句话 = 编辑入口 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md
- icon 体系一致性：同一页面 SVG icon（chevron / map-pin / clock）和 emoji（⏰🌅🎫🇨🇳）混用 = 视觉体系割裂；杂志级审美必须全站 icon 体系**单一来源**（要么全 SVG 单色描边 / 要么全 emoji 但限定主题集合）；emoji 仅限 worldEvents 国旗这类**真正承载语义**的位置，不做装饰 → 来源：research/taste-v7-线上验证-2026-04-23.md
- 彩蛋起手锚点：30-50 字短彩蛋第一句是"时间戳/小人物/具体动作"三选一（马伯庸"天宝三载""天宝十四年小吏李善德""佛讲地震有六相"三范式）还是"那是一个 XX 的时代"？后者 = 高考作文起手，50 字预算一秒耗尽 → 来源：research/taste-历史现场彩蛋+动线文风-2026-04-21.md
- 第一张图选择：地标全景明信片（铁塔全身/教堂正立面）= 通用图库；局部小切片（窗台/手/光斑/一角）= "有人在看"的视角 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md
- "我们去过"的痕迹：通篇有没有一处只能"真去过才写得出"的细节（非旺季观察/本地人小习惯/具体关门时间）？零痕迹 = 抓内容站 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md
- 动线表达层级：推荐动线是"90min/3h/半日"（时段，给选择权）> 箭头画线（视觉迁就）> "1→2→3"（顺序号，剥夺选择权）？马伯庸极限时间框架（12 时辰/15 天）本质是时段构建代入感 → 来源：research/taste-历史现场彩蛋+动线文风-2026-04-21.md
- 全局-细节嵌套导航：图片是平行列表（5 张平铺远→近→细）还是嵌套索引（1 张全局图 + 标注 3-5 锚点 + 滑进每点子图 + 回全局）？前者 Pinterest 感；后者是"走进空间"的手感 → 来源：research/taste-αβ-结构+图片模式-2026-04-22.md
- 地图 caption 动词先行：景点标注旁的一行字是"XX 清真寺：建于 1609 年，奥斯曼代表作"（名词堆砌、百科式）还是"推门抬头，21043 块瓷砖从四百年前盯着你"（动词+感官+人称）？同字数下后者信息密度 2-3 倍 → 来源：research/taste-历史现场彩蛋+动线文风-2026-04-21.md
- Survival tab 文体分层：城市级 Survival（机场/卡务/货币）口吻 = LP Safe Travel 直给式；景点级 Survival（10 美元拍照/中文套近乎）口吻 = 现场叙事式——混一起读感崩塌，UI 必须用不同 tab 或视觉权重区分 → 来源：research/taste-αβ-结构+图片模式-2026-04-22.md
- 图文交替呼吸：连续 ≥3 段纯文字没有图片打断？= 论文墙；每 200-300 字一张图 = LP 式阅读节奏，像呼吸有收有放 → 来源：research/旅游-Lonely-Planet-排版-2026-04-17.md
- 单段 viewport 上限：移动端中文单段超过 150 字（≈1 viewport 高度）= 弃读警报；超过 250 字 = 物理确认弃读（UX 实测 297 字需 2 次 swipe）；≤ 120 字 = 一屏一段最健康 → 来源：research/排版-移动端长内容自持极限-2026-04-19.md
- 对照/对开页 hinge mobile 语义保留：跨文明 / 跨时代对照卡桌面用"左 + 中央 link + 右"hinge 结构时，桌面 link 是连接句"一个 X · 一个 Y"；移动端单列堆叠后连接语义**自动失效**——必须 (a) 隐藏 / (b) 重写为单列适用句式 / (c) 改装饰条；直接保留桌面 hinge 句 = 上文重复 + 视觉上"夹中间一行小字"莫名其妙 → 来源：research/taste-CEO-对照卡-镜头诊断-2026-04-24.md
- CJK body 排版三轴：中文正文 ①字体 sans（PingFang/思源黑，不是 serif）②字号 15-16px（不是 17）③字距 0（不是继承西文 tracking）④行距 1.7？全对=读着安静；serif/过大/字距非零/低行距 任一翻车 = 阅读疲劳累加，读者弃读 → 来源：research/排版-移动端长内容自持极限-2026-04-19.md（合并自原「CJK 行距加码」）
- 参观时段叙事 > 矩阵：季节×时段用矩阵/★评分/红黄绿/"最佳时间"字样 = OTA 决策工具型，品牌立刻变机票 app；用 2-3 条编辑叙事段（"雪后第一缕光切过穹顶"）= Cereal 四季光线/LP "A Day in..."/GAC seasonal 式编辑内容型，保 taste → 来源：research/taste-深度层+参观时段-2026-04-22.md
- spot 卡独立 spotlight 层级：景点 spot 卡是否有独立 spotlight 字段（15-25 字"为什么最值得看" · 独立渲染行 · 高于 body）还是把金句混进 body？混进去 = 用户读完 body 才悟；独立 spotlight + 可走跨文明微钩范式（"比敦煌晚 171 年" / "距成吉思汗还 2 年"）= 一眼震撼、尤其对华人用户 → 来源：research/taste-v7-线上验证-2026-04-23.md

## 🟡 识别警报（fake 高级感 / 廉价感）

- 红色紧迫感三件套：红色 banner + 倒计时 + "仅剩 X" 任意出现一条 = OTA dark pattern 模板，立刻警报 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 徽章堆叠：单张卡片上贴 ≥3 个不同颜色徽章（"必去 TOP""热门""精选"...）= 廉价比拼广告 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 自动轮播 banner 占第一屏：第一屏主视觉是自动滑动 banner 而非内容首项 = 2010 门户路数 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 假社会证明数字：「X 万人想去 / Y 人正在看 / Z 人收藏」红字大号精确数字 = 焦虑营销 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 内容流广告卡：列表里出现样式略不同、内容是商品的卡片（哪怕标"广告"）= 站点要赚钱的廉价信号 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 廉价词库警报：文案出现"必去/绝美/yyds/保姆级/避坑/宝藏/天花板"任一 = 小红书营销词库，可直接复制到小红书不修改 = 掉价 → 来源：research/反向案例-小红书-旅游广告-2026-04-18.md
- "小 XX"警报：景点用"小镰仓/小京都/小冰岛/小圣托里尼"借势命名 = 没有独立审美，借别人品牌稀释原创性（全国 68 个"小镰仓"） → 来源：research/反向案例-小红书-旅游广告-2026-04-18.md
- 封面 overlay 密度：图片上 ≥3 个元素（标题文字 + emoji + 箭头/圆圈 + 数字徽章）= 信息广告牌，图片失去主角位；好图应该 overlay ≤1 → 来源：research/反向案例-小红书-旅游广告-2026-04-18.md
- emoji 花束：标题或段首堆 >2 个 emoji（🌟🔥💥📍✨ 系列）= MCN 营销模板标记，不是编辑审美 → 来源：research/反向案例-小红书-旅游广告-2026-04-18.md
- 历史短文四病警报：①"那是 XX 的时代"（套话起手）②一句内 ≥4 个年份/人名全名（考据癖）③"千年风华/岁月流转/镌刻着"（文青腔）④"承载着/蕴含着/彰显着/不仅……更……/值得一提的是"（AI-LLM 默认词库）——任一出现 = 压字数前必须先砍 → 来源：research/taste-历史现场彩蛋+动线文风-2026-04-21.md
- CJK synthetic italic 禁令：CJK 中文排版出现 `font-style: italic` / `<em>` / `<i>`（非引用书名号）= 浏览器 synthetic italic 机械倾斜中文方块字 · 头号廉价信号（Word 2003 默认）；正确做法用颜色+字重区分 em 语义 · 或只对英文 em 内联生效 → 来源：research/taste-v7-线上验证-2026-04-23.md
- 半像素字号即廉价信号：字号 15.75 / 13.5 / 11.2 类**非整数**（来自 `1.05rem` / `0.98em` 类相对计算未 round）= CSS 偷懒未 round 到整数；半像素抗锯齿模糊 → 浏览器渲染层级感被破坏 · 暴露"实现细节泄露"= 廉价信号；Cereal/LP 级要求所有字号 round 到整数 px → 来源：用户反馈-2026-04-24-CEO-视觉 2 反馈.md

## 历史淘汰记录

```
- 2026-XX-XX 淘汰「XXX」 理由：被新镜头「YYY」覆盖 / 3 轮未激活
```

- 2026-04-17 淘汰「Tap 即时反馈」理由：被「Tap 反馈分级」覆盖（更具体的分级版本）
- 2026-04-17 淘汰「跳转连续性」理由：被「共享元素过渡」+「过渡方向一致」覆盖
- 2026-04-17 淘汰「Loading 形态」理由：被「预测加载隐形」+「连续浏览不被打断」覆盖
- 2026-04-17 淘汰「视觉权重金字塔」理由：被新镜头「字重分层 > 字号跳级」+「颜色是路标不是装饰」替换（更具操作性）
- 2026-04-17 淘汰「过渡时长甜点」理由：被「复合动画 > 单轴变化」替换（时长只是一轴，复合动画覆盖更全）
- 2026-04-17 淘汰「连续浏览不被打断」理由：概念正确但偏模糊，被「动效功能检验」替换（更锐的判断标准）
- 2026-04-17 淘汰「进入退出对称」理由：与「过渡方向一致」重叠，合并保留后者
- 2026-04-17 淘汰「Touch target 工业线」理由：偏实现检查项而非 taste 镜头，被「密度递减方向」替换
- 2026-04-17 淘汰「Tap 反馈分级」理由：偏实现细节，被「字体安静检验」替换（更高层级的 taste 判断）
- 2026-04-17 淘汰「过渡方向一致」理由：被「动画属性白名单」替换（方向对称已被「复合动画」部分覆盖，白名单是更根源的纪律）
- 2026-04-17 淘汰「拇指绿区」理由：偏 dev 检查项而非 UX taste 信号，被「缓动曲线人格」替换
- 2026-04-17 淘汰「预测加载隐形」理由：实现细节型镜头，被「惊喜=多想一步」替换（更接近 UX 层面的惊喜感判断）
- 2026-04-18 淘汰「主题入口 vs 名称入口」理由：被更强版本「多轴入口平等性」替换（从"两种"升级到"≥3 种并列"，并引入 GAC 5 轴证据）
- 2026-04-18 淘汰「信任信号权重」理由：镜头范围偏窄（只讨论评分权重），被「低门槛情绪入口」替换（更根源的 taste 判断）
- 2026-04-18 淘汰「段落长度红线」理由：与「图文交替呼吸」在阅读节奏维度重叠，被「细节 zoom 能力」替换（历史定位专属）
- 2026-04-18 淘汰「Spring 软着陆」理由：过于微观、travel-web 主流程动效不会触发；腾出位置给"图片大陆可用"空白补齐
