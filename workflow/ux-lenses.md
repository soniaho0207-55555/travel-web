# UX 眼力镜头

UX 每轮 pipeline 开工前**必读**的"小本本"。研究成果浓缩在这里，让 UX 的眼睛越来越锐利。

---

## 规则（写入前必看）

- 每条 **≤1 句话**，必须是"下次看到 X 就该亮红灯"的**可观察行为**
  - ✅ 锐：`城市页第一屏：是"标题+图"还是"打动人的句子+图"？`
  - ❌ 糊：`要有呼吸感` / `图文要匹配` / `排版要优雅`
- 每条末尾加 `→ 来源：research/<文件名>.md`
- 每段上限 **15 条**，满了要淘汰最旧或最少被激活的（淘汰记到文末"历史淘汰"）
- **只有 `/research-ux` session 能写这个文件**；`/be-ux`、pipeline 的 ux-tester subagent 只读

## 🔵 长期镜头（美学 / 流畅度 / 惊喜，研究沉淀）

- Tap 即时反馈：tap 后 1 帧（~16ms）内有视觉确认（press 态/颜色/阴影/微缩）吗？没有 = "按了像没按" → 来源：research/流畅度-Linear-2026-04-16.md
- 跳转连续性：列表 → 详情是有过渡（fade/slide/transition）还是直接白屏闪烁？白屏 = Web-1.0 跳转感 → 来源：research/流畅度-Linear-2026-04-16.md
- Loading 形态：spinner 菊花（廉价）/ skeleton 骨架（合格）/ 乐观渲染（Linear 级）——看到旋转直接降一档 → 来源：research/流畅度-Linear-2026-04-16.md
- 回退保 scroll：滚 5 屏后点进详情，按回退——回到原位置 or 跳顶？跳顶 = "你刚才看的东西不重要"信号 → 来源：research/流畅度-Linear-2026-04-16.md
- 视觉权重金字塔：列表所有卡片/标签/按钮是不是"亮度阴影 icon 用色一样"？等权 = 一锅粥，眼睛扫不出节奏 → 来源：research/流畅度-Linear-2026-04-16.md
- 共享元素过渡：列表卡片图点开，是图本身"长大"成详情页（hero animation）还是新页面冒出？后者 = web-1.0 跳转 → 来源：research/产品动线-app流畅度-2026-04-16.md
- 过渡方向一致：进入用 → 滑入，返回应 ← 滑出（对称反向）；进入返回都是 fade = 失去空间方向感 → 来源：research/产品动线-app流畅度-2026-04-16.md
- 过渡时长甜点：单次 <100ms（眼睛跟不上）/ >500ms（卡顿），页面过渡 300ms 是甜点；缓动用 `cubic-bezier(0.4,0,0.2,1)` → 来源：research/产品动线-app流畅度-2026-04-16.md
- 连续浏览不被打断：瀑布流/列表往下滑时突然弹 spinner / 全屏 loading = 节奏被切；loading 只能出现在"自然停顿点" → 来源：research/产品动线-app流畅度-2026-04-16.md
- 进入退出对称：进入 X 步精心设计，退出却深埋（或反之）= 动线设计遗忘；进什么动作，应有对称反向退出 → 来源：research/产品动线-app流畅度-2026-04-16.md

## 🟢 旅游垂直镜头

- 城市页第一屏开场：纯名词标题（"巴黎"+图）= 旅游目录感；带视角/感官/时刻的一句话 = 编辑入口 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md
- 景点列表克制：一个城市列 ≥10 个 = TripAdvisor 式堆砌；≤7 个 + 每个一行"为什么是它不是别的" = Cereal 式策展 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md
- 介绍口吻：句子全是百科数据（建造年份/高度/面积）= AI 拼盘嫌疑；混入一个具体季节/时刻/非数据感官观察 = 真有人在写 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md
- 第一张图选择：地标全景明信片（铁塔全身/教堂正立面）= 通用图库；局部小切片（窗台/手/光斑/一角）= "有人在看"的视角 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md
- "我们去过"的痕迹：通篇有没有一处只能"真去过才写得出"的细节（非旺季观察/本地人小习惯/具体关门时间）？零痕迹 = 抓内容站 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md

## 🟡 识别警报（fake 高级感 / 廉价感）

- 红色紧迫感三件套：红色 banner + 倒计时 + "仅剩 X" 任意出现一条 = OTA dark pattern 模板，立刻警报 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 徽章堆叠：单张卡片上贴 ≥3 个不同颜色徽章（"必去 TOP""热门""精选"...）= 廉价比拼广告 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 自动轮播 banner 占第一屏：第一屏主视觉是自动滑动 banner 而非内容首项 = 2010 门户路数 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 假社会证明数字：「X 万人想去 / Y 人正在看 / Z 人收藏」红字大号精确数字 = 焦虑营销 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 内容流广告卡：列表里出现样式略不同、内容是商品的卡片（哪怕标"广告"）= 站点要赚钱的廉价信号 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
