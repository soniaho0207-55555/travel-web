# UX 眼力镜头

UX 每轮 pipeline 开工前**必读**的"小本本"。研究成果浓缩在这里，让 UX 的眼睛越来越锐利。

---

## 规则（写入前必看）

- 每条 **≤1 句话**，必须是"下次看到 X 就该亮红灯"的**可观察行为**
  - ✅ 锐：`城市页第一屏：是"标题+图"还是"打动人的句子+图"？`
  - ❌ 糊：`要有呼吸感` / `图文要匹配` / `排版要优雅`
- 每条末尾加 `→ 来源：research/<文件名>.md`
- 每段上限 **15 条**，满了要淘汰最旧或最少被激活的（淘汰记到文末"历史淘汰"）
- **只有 `/research-ux` session 能写这个文件**；`/be-ux`、pipeline 的 ux-tester subagent 只读

## 🔵 长期镜头（美学 / 流畅度 / 惊喜，研究沉淀）

- 回退保 scroll：滚 5 屏后点进详情，按回退——回到原位置 or 跳顶？跳顶 = "你刚才看的东西不重要"信号 → 来源：research/流畅度-Linear-2026-04-16.md
- 渐进披露层级：卡片默认只展示 1 级信息（名字+一句话+图），详情打开才见 2 级（描述/票价），主动展开才见 3 级（地图/交通）？一屏全铺 = 电话簿 → 来源：research/美学-Things-3-2026-04-17.md
- 字重分层 > 字号跳级：层级靠 weight（semibold/regular/light）+ 颜色（黑/灰）两轴区分，还是每级跳 4px 字号？后者 = 层级靠喊叫 → 来源：research/美学-Things-3-2026-04-17.md
- 颜色是路标不是装饰：内容区亮色出现几处、用在哪？彩色用于导航/分类锚点 = 功能色；彩色散落在标题/卡片/背景 = 装饰色泛滥 → 来源：research/美学-Things-3-2026-04-17.md
- 空状态有味道：列表/搜索无结果时是"暂无数据"灰字还是有设计感的留白 + 一句有温度的话？前者 = 开发占位符忘删 → 来源：research/美学-Things-3-2026-04-17.md
- 共享元素过渡：列表卡片图点开，是图本身"长大"成详情页（hero animation）还是新页面冒出？后者 = web-1.0 跳转 → 来源：research/产品动线-app流畅度-2026-04-16.md
- 动画属性白名单：过渡动画只动 transform 和 opacity？还是也动 height/width/margin？后者 = GPU 掉队、卡顿破绽 → 来源：research/惊喜-Stripe-官网-2026-04-17.md
- 复合动画 > 单轴变化：过渡是 opacity + scale + position 同时进行？只有 fade 或只有 slide = 像 PPT 切页，缺维度；缩放用 95%/90%（不是 50% 夸张跳） → 来源：research/流畅度-Apple-Music-2026-04-17.md
- Spring 软着陆：弹窗/卡片展开末尾有微回弹（spring）还是线性急停？急停 = 机械感；微弹 = "有重量的东西落下来"的物理感 → 来源：research/流畅度-Apple-Music-2026-04-17.md
- 动效功能检验：每个动画能一句话回答"这在告诉用户什么"（来源/去向/层级）？回答不了 = 装饰动效，删掉比留着好 → 来源：research/流畅度-Apple-Music-2026-04-17.md
- 缓动曲线人格：全站动效用同一条自定义 cubic-bezier 还是每处随手写 ease / linear？后者 = 没有动效人格，每个动画像出自不同团队 → 来源：research/惊喜-Stripe-官网-2026-04-17.md
- 密度递减方向：从导航区到内容阅读区，信息密度是递减的（导航紧凑/阅读松弛）还是全页统一密度？统一密度 = 一刀切，没有功能区分 → 来源：research/美学-Bear-2026-04-17.md
- 字体安静检验：读完一屏内容后你记得字体长什么样吗？记得 = 字体太抢戏；忘了 = 字体在正确地支撑内容（Bear Sans 的设计目标：sit silently） → 来源：research/美学-Bear-2026-04-17.md
- 惊喜=多想一步：列表/详情/返回流程里有没有一处"它居然记得我上次做了什么"（访问过标记、上次阅读位置、偏好保留）？零处 = 功能齐但没人格 → 来源：research/惊喜-Stripe-官网-2026-04-17.md
- 图片占位无抖动：图加载前用 `aspect-ratio` 撑开高度？没设 = 加载完图后下方内容"往下跳" = layout shift 破绽 → 来源：research/产品动线-H5装native-2026-04-16.md

## 🟢 旅游垂直镜头

- 城市页第一屏开场：纯名词标题（"巴黎"+图）= 旅游目录感；带视角/感官/时刻的一句话 = 编辑入口 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md
- 景点列表克制：一个城市列 ≥10 个 = TripAdvisor 式堆砌；≤7 个 + 每个一行"为什么是它不是别的" = Cereal 式策展 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md
- 介绍口吻：句子全是百科数据（建造年份/高度/面积）= AI 拼盘嫌疑；混入一个具体季节/时刻/非数据感官观察 = 真有人在写 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md
- 第一张图选择：地标全景明信片（铁塔全身/教堂正立面）= 通用图库；局部小切片（窗台/手/光斑/一角）= "有人在看"的视角 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md
- "我们去过"的痕迹：通篇有没有一处只能"真去过才写得出"的细节（非旺季观察/本地人小习惯/具体关门时间）？零痕迹 = 抓内容站 → 来源：research/旅游-Cereal-Magazine-2026-04-16.md
- 多轴入口平等性：首页有几种平行进入路径？只有"先选地点"1 条 = 要求用户已有目的地；≥3 种平行维度（时期/主题/情绪/大洲/人物）= 灵感驱动友好，GAC 用 5 轴并列（Artists/Mediums/Movements/Events/Places） → 来源：research/旅游-Google-Arts-Culture-2026-04-18.md
- 图片排序叙事：详情页多张图是随机排列还是有叙事顺序（远景→建筑→细节→人→安静角落）？无序 = Pinterest 碎片；有序 = 镜头推移在"走进"这座城 → 来源：research/旅游-Airbnb-Kyoto-2026-04-17.md
- 色彩聚焦度：页面亮色（非黑灰白）出现几处？≥3 处不同颜色高亮 = 视觉噪音注意力分散；仅 1 处（CTA / 关键标签）= Airbnb 式聚焦 → 来源：research/旅游-Airbnb-Kyoto-2026-04-17.md
- 低门槛情绪入口：首页有没有一个"不需要专业知识就能点"的入口（颜色/氛围/心情/季节）？有 = GAC 式普惠（"What's your favorite color?"）；没有 = 只服务已有知识储备的用户 → 来源：research/旅游-Google-Arts-Culture-2026-04-18.md
- 图文交替呼吸：连续 ≥3 段纯文字没有图片打断？= 论文墙；每 200-300 字一张图 = LP 式阅读节奏，像呼吸有收有放 → 来源：research/旅游-Lonely-Planet-排版-2026-04-17.md
- 细节 zoom 能力：历史地标 / 文化景点主图能放大到看清砖缝题字 / 碑文 / 壁画细节吗？还是只有一张明信片视角？后者 = "我没真去过"的气质；前者 = 历史定位的金标准（GAC Gigapixel 模式） → 来源：research/旅游-Google-Arts-Culture-2026-04-18.md
- 间距有韵律：段间距/章节间距/图片上下留白是同一基础单位的整数倍（如 8px/16px/32px/64px）？还是随手写的不同值？后者 = 排版没有节拍 → 来源：research/旅游-Lonely-Planet-排版-2026-04-17.md
- 移动端图片全宽 bleed：图片突破内容 padding 延伸到屏幕边缘？是 = 小屏杂志冲击力；否（图片也被 padding 框住）= 内容被"关在盒子里" → 来源：research/旅游-Lonely-Planet-排版-2026-04-17.md
- CJK 行距加码：中文内容行距是 1.7-1.8 还是沿用西文 1.5？后者 = 中文字挤成一团，每个字符都是完整语义单元，需要更多视觉呼吸 → 来源：research/美学-Bear-2026-04-17.md
- 续流 overlay 保上下文：看完一个城市/景点后有没有"where next"式相关推荐，同时保持上下文（暗化背景而非全屏跳转）？没有 = 阅读断在尽头；有 = 续流丝滑且不丢认知锚点 → 来源：research/旅游-Google-Arts-Culture-2026-04-18.md

## 🟡 识别警报（fake 高级感 / 廉价感）

- 红色紧迫感三件套：红色 banner + 倒计时 + "仅剩 X" 任意出现一条 = OTA dark pattern 模板，立刻警报 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 徽章堆叠：单张卡片上贴 ≥3 个不同颜色徽章（"必去 TOP""热门""精选"...）= 廉价比拼广告 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 自动轮播 banner 占第一屏：第一屏主视觉是自动滑动 banner 而非内容首项 = 2010 门户路数 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 假社会证明数字：「X 万人想去 / Y 人正在看 / Z 人收藏」红字大号精确数字 = 焦虑营销 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 内容流广告卡：列表里出现样式略不同、内容是商品的卡片（哪怕标"广告"）= 站点要赚钱的廉价信号 → 来源：research/反向案例-OTA廉价感-2026-04-16.md
- 廉价词库警报：文案出现"必去/绝美/yyds/保姆级/避坑/宝藏/天花板"任一 = 小红书营销词库，可直接复制到小红书不修改 = 掉价 → 来源：research/反向案例-小红书-旅游广告-2026-04-18.md
- "小 XX"警报：景点用"小镰仓/小京都/小冰岛/小圣托里尼"借势命名 = 没有独立审美，借别人品牌稀释原创性（全国 68 个"小镰仓"） → 来源：research/反向案例-小红书-旅游广告-2026-04-18.md
- 封面 overlay 密度：图片上 ≥3 个元素（标题文字 + emoji + 箭头/圆圈 + 数字徽章）= 信息广告牌，图片失去主角位；好图应该 overlay ≤1 → 来源：research/反向案例-小红书-旅游广告-2026-04-18.md
- emoji 花束：标题或段首堆 >2 个 emoji（🌟🔥💥📍✨ 系列）= MCN 营销模板标记，不是编辑审美 → 来源：research/反向案例-小红书-旅游广告-2026-04-18.md
