# 用户反馈 · CEO 4 条反馈圣索菲亚诊断

**对象**：https://soniaho0207-55555.github.io/travel-web/istanbul-exp/index-h5.html#/city/istanbul/landmark/0
**身份**：28yo taste-driven，线上实测
**说明**：CEO 4 条是症状观察，逐条给【症状/诊断/药方/理由】。反馈 1 和反馈 3 **同一个病**，明确拆开说。

## 反馈 1 · 读一读冗长、无加粗、无重点、无图标

**【症状】** 读一读 4 段纯文字墙（是什么 / 为什么独特 / 跨文明 / 现场细节），灰 15px / 400 weight 一路铺到底。

**【诊断】** 症状诊断正确。**根因是"没有视觉节奏"**——
- 体检结果：字体 sans / 字号 15px / 字距 0 / 行距 ~1.7 —— CJK 正文三轴全合格
- 但通篇**零加粗金句 / 零 inline icon / 零对照卡片 / 零小图插入**
- Lonely Planet 每 200-300 字插一个 anchor；这里 771 字一片灰

**【药方】** 3 层叠加，不必全做：
1. **金句加粗**：每段拎 1-2 句升到 17px + 加粗 + 换色（如 #E8D4B0 金沙）。例："让整座圆顶看起来像**浮在空中**" / "一座建筑教会了另一个帝国" / "都在对抗同一个时代的普遍困惑：**神是否仍在**"。读者即使跳读也能抓住骨架
2. **"跨文明"段改对照卡**：这段本质是 A vs B 的结构（查士丁尼 40 扇窗 vs 北魏石窟），**应该是左右两栏 card**，不是散文。同字数容量 / 并排对照 / 中央 vs 字样连接。这是最值得改的一处
3. **inline icon 小节分隔**：每 section 标题前加一个 16px 线性 icon（📖 是什么 / ✨ 为什么独特 / 🌍 跨文明 / 🔍 现场细节）—— 注意**不是 emoji**，是细线 SVG。Emoji 会滑进"小红书 MCN 花束"警报

**【理由】** 我真实读的时候，"跨文明"段读到 2/3 开始想跳过（一段太长且结构是"中国这样…拜占庭那样…都在…"的平行散文）。加粗金句后我会"扫读抓骨架"而不是弃读。对照卡是把 200 字扁平散文变成 2×100 字结构化信息的最优解。

## 反馈 2 · 深一步字体不和谐

**【症状】** CEO 说"字体不和谐"—— 模糊感受。

**【诊断】** 症状对。**4 个具体技术病**：

1. **Pullquote 用 italic + 中文无真斜体** ← 头号元凶
   pullquote-text 设了 `font-style: italic`，但 system sans 和 PingFang SC 对中文字符**没有真正的斜体字形**，浏览器做 synthetic italic（机械倾斜）—— 结果"石头从拜占庭地下引上来的管道输送过"字形变形、字重发虚、边缘不干净。且同句里 cite "Cyril Mango, *Hagia Sophia*" 的英文斜体是**真 italic**，中英两种斜体质量拼一起 = 视觉不和谐的根源
2. **Pullquote 区分度不足**：16px vs body 15px，差 1px 肉眼看不出。按杂志级该 ≥18px / 或左侧粗竖线 / 或颜色完全分离
3. **Eyebrow "工艺 / 争议" 10px + 1.8px letter-spacing**：字号太小在深色上勉强可读；CJK 不需要这么大的字母追踪，1.8px 让字与字距太散。降到 11-12px + 0.5px
4. **Layer label 字号不一**：深一步 20px / 读一读 20px / 出发前 18px —— 三层级同级，字号该统一。出发前字号小 2px 让层级感觉被降级

**【药方】**
- pullquote: 去 italic → 左侧 3px #B53A1F 竖线 + 18px + 金沙色（Cereal / NYT / GAC 的做法）
- eyebrow: 11px + letter-spacing 0.5px + 色块背景保持（现在的 rgba(181,58,31,0.08) 做得对）
- layer label: 统一 20px（或都 18px，但统一）
- cite 保持 11px 但建议加一条 hairline 上横线分隔

**【理由】** 假斜体是第一眼就能察觉的廉价感——杂志排版师最在意的细节。Cereal / Monocle / Apertura 没一家敢对中文用 synthetic italic。这条改完"字体不和谐"直接消失。

## 反馈 3 · 旅行前 tab 太长，拆"出发前"成独立 tab，改名"旅行前的沙发时间"

**【症状】** 旅行前 tab 总高 5400px，确实太长。

**【诊断】** **CEO 误诊（CEO 已自认可能误诊）**。

- 假根因："tab 内容多 → 拆 tab"。但拆成 4 个 tab = 信息堆砌的 TripAdvisor 路数，违反我们的 Jobs 克制哲学
- **真根因**：3 层架构已经在那儿（读一读 / 深一步 / 出发前），但**折叠 UX 没兑现**——深一步默认折叠（✅），但**读一读和出发前都默认展开**，用户一进来看到的是 5400px 灰墙，不知道有分层可跳
- 和反馈 1 **同一个病**：信息密度高 + 无视觉导航锚点

**【药方】** 反对拆 tab，推荐：
1. **顶部加章节锚条**（sticky，离 tab bar 下 0 pt）：
   `📖 读一读（沙发）  ·  🔍 深一步  ·  🧳 出发前`
   点击瞬移 + 当前位置高亮。像 NYT / GAC / Apple 长文页
2. **读一读"现场细节" 仍默认展开**（这是第一印象锚点）；**出发前默认折叠**（和深一步一致）—— 3 层里只 1 层默认展开，2 层 tap-to-open
3. **CEO 改名"旅行前的沙发时间"**：语义只准确描述读一读，盖不住深一步（严肃长读）和出发前（票务实用）。**不改名**，让 3 层各自的 sub-label（"沙发态·故事" / "历史背景·点开读" / "票务·实用手册"）继续工作

**次选**（如 CEO 坚持拆 tab）：拆出"实用"tab（票务 + tips + 注意事项）——这至少比改名有效。但本质还是堆 tab，**不推荐**。

**【理由】** 锚条 + 折叠是 NYT 深度长文 / Bear 阅读器 / Apple 新闻的标准做法。28yo 用户进长读页**期待**有锚点导航，不会为"太长"抱怨——只会为"长且看不到结构"抱怨。反馈 1 的金句加粗 + 反馈 3 的锚条折叠 = **同一套视觉语言**，一起做。

## 反馈 4 · 旅行中图直观但无 spotlight 讲为什么值得看

**【症状】** CEO 说"没有表明每个地点为什么值得看"。

**【诊断】** **症状偏，实际已有 hook 但视觉太弱**。

- 实测 3 个 spot 的 body 文字（都 1-2 句）：
  - 许愿柱："石柱一千年都在出汗。把拇指塞进凹洞转一圈——**从东罗马主教到你，队伍没断过**。"
  - Dandolo："1204 年是他带十字军洗劫这里。**你正踩在毁它的人之上**。"
  - 书法圆盘："8 块书法圆盘挂在头顶，**每块 7.5 米比两个成人还高**。"
- **hook 句都在那里**，而且都是"震撼级"—— Dandolo 那句我读的时候起鸡皮疙瘩。对华人友好的比例化（"比两个成人还高"）也到位
- **真问题**：hook 句全部是 body 15px / 灰色 / 400 weight 普通段，没加粗、没变色、没 pullout。**震撼句被埋了**
- **附加问题**：3 个 spot 完全平级并列，没有"最值得"的**策展稀缺感**

**【药方】**
1. **hook 句视觉突出**：每个 spot 的那一句震撼的话（"队伍没断过" / "踩在毁它的人之上" / "比两个成人还高"）—— **加粗 + 17px + 金沙色**。相当于把它从 body 提升到 pull-quote 一档
2. **编辑策展"最值得"标记**：3 个 spot 里选 1 个加**"编辑推荐"细小徽章**（右上角，深绿或深金，不是红色）。例：许愿柱是许愿 + 1200 年 + 物理学解谜三合一，最值得。这是 Airbnb "Guest favourite" / Cereal "Editor's pick" 的做法
3. **不要加字**：CEO 明确说"字不要多"—— hook 句仍 1 句，不加新文案

**【理由】** 读者在旅行中 tab 目的是**速览 + 挑看点**。现在 hook 存在但被平视层级淹没 = 读者不知道自己读了个爆点。视觉突出后"队伍没断过" / "踩在毁它的人之上"这样的句子会**留在 48 小时记忆里**—— 那是 Apple / GAC / Cereal 的核心功夫。
