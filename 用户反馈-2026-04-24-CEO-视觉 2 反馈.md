## 反馈 1 · 故事栏目"字多 / 大小不一 / 廉价"

### 诊断 1.1 · 哪段字最多？

**不是单段字多，是元素密度爆表**。

实测单段字数（均低于 150 字 viewport 上限硬规）：
- 是什么：115 字
- 为什么独特：121 字
- 跨文明：101 字（两栏 + 中间对照句）
- 现场细节：186 字（5 slide caption 累计）

**真根因**：从 Hero 到现场细节短短 2 屏里，**同时存在 10+ 个不同字号的文字元素**：

| # | 元素 | 字号 | 粗细 | 字族 |
|---|---|---|---|---|
| 1 | lm-name "圣索菲亚大教堂" | 28px | 700 | Playfair |
| 2 | lm-era "拜占庭·537年" | **11.2px** | 400 | **Courier Prime** |
| 3 | hookShort | 20px | 500 | Playfair |
| 4 | tab "沙发时间" | 15px | 500 | sans |
| 5 | anchor "故事" | 13px | 400 | sans |
| 6 | layer-label | 20px | 700 | Playfair |
| 7 | layer-sub | 11px | 400 | sans |
| 8 | section-title "是什么" | **15.75px** | 600 | sans |
| 9 | body p | 15px | 400 | sans |
| 10 | cross-side-title | 15px | 600 | Playfair |
| 11 | cross-side-bullets | **13.5px** | 400 | sans |
| 12 | slide-caption | 13px | 400 | sans |
| 13 | carousel-hint | 11px | 400 | sans |

**字号清单**：28 / 20 / **15.75** / 15 / **13.5** / 13 / **11.2** / 11 → **8 级字号**（其中 3 个半像素小数）。违反"字号台阶 ≤ 3 级" lens **爆表 2.5 倍**。

### 诊断 1.2 · "廉价"从哪里来（4 个具体元凶）

**元凶 1 · lm-era "拜占庭·537年" 用 Courier Prime 等宽 + warm gold #C9963A**
- 等宽字 mono 在整站唯一一处，格格不入。Courier 用得好（Apple SF Mono 做 meta）是气质；这里 11.2px + 金黄橘 = "90 年代黄页广告副标"
- Apple / Cereal / Monocle 的 era/date 用 serif 细体或 sans caps，**从不用 Courier**

**元凶 2 · section-title "是什么 / 为什么独特 / 跨文明 / 现场细节" 用 15.75px + #D4A574**
- 颜色 #D4A574 是**偏暖的甜金**——接近小红书 MCN 标题色。不是 Cereal 的冷金
- 15.75px 有小数 → 半像素抗锯齿模糊 → **实现细节泄露**是廉价信号（整数字号是基本工程纪律）
- 4 个标题**文风不统一**："是什么"（口语问句）/ "为什么独特"（口语问句）/ "跨文明"（名词）/ "现场细节"（导购词）= 混搭到了离谱

**元凶 3 · carousel-hint "← 横滑看下一片 →"**
- ASCII 箭头 + "横滑看下一片" 6 字教程式口语提示 = 旅游穷游攻略页的标配信号
- Cereal / LP / Apertura 只用 dot indicators（● ○ ○ ○）+ 可选手势 icon，**从不写"横滑看"**
- 这类 "教用户怎么操作" 的文字 hint 本身就是"我不自信我的交互能被理解" 的露怯

**元凶 4 · 字号半级跳（15.75 / 13.5 / 11.2）**
- `15.75 vs 15 差 0.75px` / `13.5 vs 13 差 0.5px` —— 肉眼几乎看不出，但**破坏了层级感**
- 这种数值来自 `font-size: 1.05rem` / `0.98em` 类相对计算，**是代码偷懒没 round 到整数**的表现

### 药方 · 字号收敛到 3 级（强制硬规）

```
L1 标题 · 20-28px
  lm-name 28 / hookShort 20 / layer-label 20

L2 小节 · 15-16px
  section-title 16（非 15.75）/ body p 15 / tab 15 / cross-side-title 15

L3 caption · 11-13px
  anchor 13 / era 13（非 11.2）/ sub 11 / cross-bullet 13（非 13.5）/ slide-caption 13 / hint 11
```

**废除**：15.75 / 13.5 / 11.2 三个半级跳（→ round 到 16 / 13 / 13）。

### 药方 · 廉价感 4 病修方

| 病灶 | 当前 | 改为 |
|---|---|---|
| lm-era 字族 | Courier Prime 11.2px #C9963A | Playfair 细体 13px #E8D4B0 金沙 · 或 sans caps |
| section-title 颜色 | #D4A574 warm gold | #E8D4B0 金沙（统一到 hookShort/layer-label）|
| section-title 文风 | 是什么/为什么独特/跨文明/现场细节 | **都 2 字诗化**："空间 / 结构 / 对照 / 现场" · **或都问句**："是什么 / 特别在哪 / 同代文明 / 去看什么" |
| 字号小数 | 15.75 / 13.5 / 11.2 | round 到整数 |
| carousel-hint | "← 横滑看下一片 →" 文字 | 删掉 · 换 dot indicators（● ○ ○ ○ ○）· 滑 1 次后消失 |

### PRD 硬规建议（CEO 定调"以后不能有"）

```markdown
## §R-08 字号与字族台阶硬规（2026-04-24 起）

- 单页面 / 单 tab 内字号档位 **≤ 3 级**（L1/L2/L3）：
  - L1 标题 · 20-28px
  - L2 小节 · 15-16px
  - L3 caption · 11-13px
- **严禁半像素字号**（非整数值如 15.75 / 13.5 / 11.2）· round 到整数
- 同档位内字族 **≤ 2 种**（杜绝 3 家 sans/serif/mono 混搭）
- **Mono 等宽字体**（Courier / SF Mono）仅限**数字/代码/坐标类元数据**，**不用于时代名/标题/正文**
- section-title **文风必须统一**（全问句 / 全名词 / 全诗化，选一种，不混搭）

**违反 = QA 视觉审 FAIL · 不进 main**
```

## 反馈 2 · 现场细节"怪怪的"

### 诊断 2.1 · 为什么感觉怪（5 个真根因）

**根因 1 · 阅读节奏突然断裂**
前 3 section（是什么 / 独特 / 跨文明）都是**文字段落为主**，到现场细节**突然切换成横滑 carousel + 图 dominant**。读者被训练的节奏突然换频道。

**根因 2 · 交互模式和气质冲突**
横滑 carousel 是 **Instagram / 小红书爽刷模式**——信息是扁平并列的 "X 张 / Y 条"。这和前面 Cereal-like 的**长读严肃气质**硬冲突。Cereal / LP Pocket **从不在长读内用横滑**。

**根因 3 · 叙事层级被扁平化**
5 张图本来是递进：仰拍总览 → 450 年覆盖史 → 2009 揭开 → Viking 涂鸦 → 空间感。横滑把**层级递进**压成**平行展品**——变博物馆展柜，不是编辑视角。

**根因 4 · caption 风格倒置**
caption 文字是"新闻配图注解"质感（"1453 年奥斯曼征服后灰泥覆盖了 450 年"）—— **图为注、文为主**的倒错。现场细节应该是 **图为主体、caption 空间锚定**（"二楼西北角" / "主穹顶 12 点方向"）而不是背景介绍。

**根因 5 · "← 横滑看下一片 →"教程式 hint**
（见反馈 1 元凶 3。作为反馈 2 同步触发）

**Bonus 根因 6**：slide 5 caption **"这里躺过的不只是皇帝的仪仗队。"** —— **没图索引**、意图隐晦（指代墓葬？扶柩队伍？）、**第五张图本体到底是什么读者不知道**。这种"悬念 caption"在杂志里要配"空镜+伸缩想象"，当前图又是穹顶全景——图文彻底脱钩。

#### 🟢 候选 A ·「空间锚点 grid」（UX 推荐）

> 像 Cereal / Apertura / Monocle "A Closer Look" 板块

```
┌──────────────────────────────┐
│ 现场（或改："去现场看什么"）  │
│                              │
│ ── 抬头 ──────────────────── │
│ [穹顶天使图 · 16:9]          │
│ 穹顶四角 · 9 米直径           │
│ 翅膀罩住整个大殿              │
│                              │
│ ── 墙上 ──────────────────── │
│ [灰泥图 · 16:9]              │
│ 1453 年后 450 年覆盖          │
│ 2009 年剥掉西北一面           │
│                              │
│ ── 角落 ──────────────────── │
│ [Viking 图 · 16:9]           │
│ 西南柱 · 13 世纪卢恩文        │
│ "Halfdan 来过"               │
│                              │
│ （继续：脚下 / 正前 / 二楼）  │
└──────────────────────────────┘
```

**特点**：
- **eyebrow 用空间方位锚**（抬头 / 墙上 / 角落 / 脚下 / 正前 / 二楼）· 是**只有真去过才会给的向导感**
- 图 60% + 文字 2 行 40%，每条中间 hairline 分隔
- **上下流式，不横滑** · 阅读节奏和前面 3 section 一致
- 每条 **caption 2 行**（空间位置 + 事实）· 不是长段介绍
- 5-6 条顺序是**游客走动线**（进门抬头→转头看墙→找角落→等等），**隐式教游览顺序**

**优点**：
- 和 Cereal 严肃杂志审美最贴
- 实现成本**最低**（只是把 carousel 换成 vertical list 加 eyebrow）
- "现场细节"改名为"现场"或"去看什么"更直接
- 保留图的视觉重量，又没有"爽刷"廉价

**缺点**：列表稍长，需要折叠机制（默认展开 3 条 + 点"看更多"展开剩余）

```
│                              │
│ [详情抽屉 · 当前 ①]           │

│ [天使特写图]                 │
│ 穹顶四角 · 翅膀罩整殿          │
└──────────────────────────────┘
```

**特点**：
- 一张**全景大图** + 5 个红圈编号（和旅行中 tab 的红圈系统一致）
- 点圆圈展开下方详情卡（图 + 2 行说明）
- 空间关系**一目了然**（游客知道"去了抬头找 ①"）

**优点**：
- 和旅行中 tab 的红圈视觉语言**打通**——一套 icon 系统
- 互动揭示有探索感 · 适合 iPad 放大体验
- **空间感 > 候选 A**

- 实现成本高（需要全景图 + 热点坐标 + 抽屉交互）
- 全景图 Wikimedia 不一定能找到合适角度
- 小屏 375px 宽下红圈密集会碰撞

```

│ 穹顶仰视全景（9 米天使隐约可见）│

  去现场看什么

  · 二楼西北角
    流泪石柱 · 凹洞 1200 年出汗至今
```

**特点**：
- 1 张 hero 大图 + 文字 bullet 清单
- 每条 2 行：空间方位 + 一句事实
- **图少 · 文字主导**

**优点**：
- 最简洁 · 阅读节奏和前面段落完全一致
- **没有任何横滑** · 没有 caption 混乱
- LP Pocket / Wallpaper Guide 的典型做法

**缺点**：
- 视觉惊喜度降低（只 1 图）
- 读者看不到细节真容（撒拉弗天使脸是这页最抓人的图之一）

#### UX 推荐 · **候选 A（空间锚点 grid）**

理由：
1. **空间方位锚**是"真去过"的向导感 —— 契合 🟢 "我们去过"的痕迹 lens
2. 阅读节奏和前 3 section 一致（垂直流式）—— 不断裂
3. 实现成本**最低**（把 carousel 改 vertical list + eyebrow · CSS 级改动）
4. **保留图的主角感** · 又消除"爽刷"廉价
5. 每条 2 行 caption**隐式教游览动线**

候选 B 更酷但实现重 + 图源风险 · 候选 C 过简损失视觉 · **A 平衡最好**。

### B 方案规格细化

#### 1. 全景底图（PM 采图职责）

**对象**：圣索菲亚**内殿全景**（不是外立面 / 不是平面图，是站在大厅地面或二楼回廊**仰拍 / 斜视**的全景）

**优先级**：
1. **首选**：现有 v3.5-exp slides 里 `hagia-A2-panorama-crop.jpg` 或 `hagia-A5-panorama-full.jpg`——已采、已合规、CEO/UX 二轮已过审。**复用即可**，不重新采
2. **备选**：Wikimedia "Hagia Sophia interior panorama" 类目里的更高分辨率版（≥1920px 宽，确保红圈 zoom 后仍清晰）
3. **底线**：禁止用平面 Grundriss（已在 maps tab 用过 · 重复）
