## 1. 真机第一感受

**真的丑。CEO 没说错**。

我打开圣索菲亚页 · 滑过"是什么/为什么独特" · 到"跨文明"那一刻 —— 第一反应是"这是什么 90 年代竖排诗页 ?"。具体崩在 4 处：

### 崩点 1 · **金句"石"字孤字断尾**（最致命）

中间那行 "一个让神飘空中 · 一个让神埋岩石"（共 14 字）在 202px 宽容器里**被强制换行成 2 行**：

```
一个让神飘空中 · 一个让神埋岩
石                              ← 孤字！
```

**最后一个"石"字单独占一行** —— 这是排版禁忌"孤字断尾"（中文叫"挂尾" / 西文叫 widow）。
LP / Cereal / Monocle 的中文排版**绝不允许**单字甩尾 —— 这是初稿到清稿之间最该被抓的廉价信号。

成因：12px 字号 + letter-spacing 0.72px（**中文不该加字距**）+ padding 8px → 实际可用宽度只够塞 13 字 → 第 14 字"石"落第二行。

### 崩点 2 · **金句视觉权重过低**

实测金句样式：
- 字号 **12px**（比 bullet 13.5px 还小）
- 颜色 **#8A7F6D**（暖灰 · 在暗色背景上几乎看不见）
- weight **400**（普通粗细）

→ 这是"金句"还是"段尾注释"？读者扫一眼直接跳过。
→ 12px 是 caption 字号档位，金句应该在 17-20px 字号档。
→ 颜色应该是金沙 #E8D4B0（和 hookShort/layer-label 同色系），不是暖灰。

### 崩点 3 · **两个 card 没有视觉差异**

拜占庭 card 和北魏 card —— 同样的 bullet 样式 / 同样的 #C9B89A 金标题 / 同样的间距。
→ "对照"的精髓是"两边气质不同"。Cereal 的对照排版会让一边偏冷一边偏暖，或一边底色微深、一边微浅。
→ 现在两边视觉**完全相同**，对照感消失，只剩"两个一样的盒子上下叠"。

### 崩点 4 · **金句紧贴下一个 card 标题**

"一个让神飘空中·一个让神埋岩石" 后面紧接 "北魏·同一时代" 标题，**间距只有 ~12px**。
→ 金句被吃进北魏 card 头里 · 视觉上像北魏的副标
→ 应该有 hairline 分隔线（上下各一条 1px #4A4135 hairline），把金句**抬到顶点位置**

**总评**：当前对照卡是整个故事 tab 里最丑的一段。CEO 强情绪"改掉"完全合理。

## 2. 三个候选呈现形式

### 🟢 候选 A ·「上下堆叠 + 金句金线独立」（UX 推荐）

```
┌─────────────────────────────┐
│ 跨文明                      │
│                             │
│ ┌─────────────────────────┐ │
│ │ 拜占庭 · 537 年          │ │
│ │ · 查士丁尼建成圣索菲亚    │ │
│ │ · 40 扇窗让神飘在空中     │ │
│ │ · 石柱 + 金箔马赛克      │ │
│ └─────────────────────────┘ │
│                             │
│ ──────── ✦ ────────         │
│ 让神飘在空中 · 让神埋进岩石   │
│ ──────────────────          │
│                             │
│ ┌─────────────────────────┐ │
│ │ 北魏 · 同一时代           │ │
│ │ · 开凿龙门石窟            │ │
│ │ · 让神埋进岩石            │ │
│ │ · 石像 + 石窟庙堂         │ │
│ └─────────────────────────┘ │
└─────────────────────────────┘
```

**改动**：
1. **金句独立成行 · 上下 hairline 包夹**（1px #4A4135） · 中间 ✦ 装饰
2. **金句字号 12px → 18px** · weight 400 → **600** · 颜色暖灰 → **金沙 #E8D4B0**
3. **金句句式改对仗**："让神飘在空中 / 让神埋进岩石" · 删掉廉价"一个...一个..."句式
4. **letter-spacing 0.72 → 0**（中文不要字距）—— 解决孤字断尾
5. **两 card 底色微差**：拜占庭 card bg 微冷 (rgba(180,200,220,0.04)) · 北魏 card bg 微暖 (rgba(220,180,140,0.04)) —— 5% 透明度即可，不抢戏

**优点**：
- 实现成本**最低**（CSS 级改动，无需采图）
- 金句**真正成为顶点**（独立行 + 大字 + hairline 包夹）
- 两 card 微差异打破"两盒子叠"的雷同
- 兼容现有 PRD 数据 schema

**缺点**：仍是文字主导 · 视觉冲击力中等

### 🟡 候选 B ·「横向对账表」（杂志严肃风）

```
┌─────────────────────────────┐
│ 跨文明                      │
│                             │
│            拜占庭 │ 北魏     │
│            537   │ 同代     │
│ ───────────────────────────│
│ 建筑       圣索菲亚 │龙门石窟 │
│ ───────────────────────────│
│ 媒材       石柱马赛克│ 石像庙堂│
│ ───────────────────────────│
│ 姿态       神飘空中  │ 神埋岩石│
│ ───────────────────────────│
└─────────────────────────────┘
```

**特点**：
- 4-5 行对账表 · 行首是维度（建筑 / 媒材 / 姿态 / etc.）
- 中间一条竖向 hairline 分隔
- "神飘空中 / 神埋岩石" 是表格**最后一行的姿态对照** —— 自然成为表底金句
- 顶部两栏头是 "拜占庭 537 / 北魏 同代"

**优点**：
- 最像 Monocle / Wallpaper 的"对账格" · 严肃杂志感顶级
- 信息密度高 · 看一眼就能 grok
- 不需要采图

**缺点**：
- 表格在 375px 窄屏上每格仅 ~140px · 文字易溢出
- 维度名（建筑/媒材/姿态）需要 PM 重新提炼 · 当前 bullet 散文不直接套用
- 移动端表格 taste 风险高（容易像 OTA 价格对比表）—— 需要 hairline 而非 border

**特点**：
- 两个**图片卡上下堆叠**（375px 下天然堆叠 · 768px+ 才并排）
- 每卡顶部一张代表图（圣索菲亚穹顶 + 龙门石窟）
- 文字 3 行 bullet · 底部 1 行金句
- 没有"中间金句"段 —— 金句拆成两半分别落在两 card 底部

**优点**：
- 视觉冲击力**最强** · 像 Cereal 跨文化对比专题

- 两 card 因为图本身气质天差（金色穹顶 vs 灰岩石窟）· 自然形成对照感

**缺点**：
- **需要 PM 采图**：龙门石窟图需要从 Wikimedia 找 + 合规审 · 1-2 个 session
- 实现成本最高 · 候选 B 落地节奏可能延期
- 圣索菲亚的图可以复用现有 hero · 龙门图是新的

## 3. 中间金句视觉权重判断

**强烈建议：升 18px + 上下 hairline + 金沙色 #E8D4B0**。

**理由**：

| 维度 | 当前 v7.0 | 应该改为 |
|---|---|---|
| 字号 | 12px（caption 档）| **18px**（金句档）|
| 粗细 | 400 | **600** |
| 颜色 | #8A7F6D 暖灰 | **#E8D4B0 金沙**（同 hookShort 色系）|
| 字距 | letter-spacing 0.72px | **0**（中文不要字距）|
| 装饰 | 无（紧贴上下 card）| **上下 1px #4A4135 hairline** · 中间 ✦ 或 — 装饰 |
| 句式 | "一个让神飘空中·一个让神埋岩石" | **"让神飘在空中 · 让神埋进岩石"**（去廉价"一个...一个..."）|
| 行宽 | container 202px · 强制 2 行 | container ≥ 280px 单行 · 或允许 2 行但**禁单字尾**|

**判断依据**：
- "金句"是 **L1 标题档** 的修辞强度（"做过一千年教堂，五百年清真寺，现在在做第三件事" 是 hookShort 20px 金沙 · 那是金句的标杆）
- 当前对照卡的"金句"是 **L3 caption 档**字号、weight、颜色 —— 视觉权重错置
- 金句需要**独立空间** —— 上下 hairline 是杂志排版"breathing room"的硬规

## UX 推荐 · **候选 A**

理由：
1. **实现成本最低**（CSS 级改动 · 无新数据 · 无新图源）
2. **直接修掉 4 个崩点**（孤字断尾 / 金句弱 / 雷同盒子 / 紧贴）
3. v7.3 PRD 落地节奏不会因为采图延期

候选 C **作为下一轮迭代候选**保留 —— 等 PM 有空采龙门图源后可以从 A 升级到 C。
