# research: taste - 深度层+参观时段 v5 - 2026-04-22

## 反馈 1 · 深度层 taste 标准 · worldEvents 够深吗？

**不够**。worldEvents 是"同期横向扫 N 条"——**外围层**不是深度层。

三参照物的深度形态：

| 对象 | 深度形态 | 特征 |
|---|---|---|
| Cereal sidebar | 一个具名工匠/一种工艺/一段语录 | 单一焦点 + 下沉到末端 |
| LP secondary box | "Historical lowdown" / "X 年前在这发生了 Y" | 时间锚定 + 单一事件 |
| GAC deep dive | 一幅画 2000+ 字 + gigapixel | 单主题 + 末端穷尽 |

**共性：深度 ≠ 多信号叠加，深度 = 单一焦点穷尽到末端**。worldEvents/meanwhile/relatedFigure 三条横向并列 = 目录感，没一条钻深。

**v5 深度层本就偏薄**。建议补 4 类字段（**择一轮换，非全塞**，克制如 Cereal 每期只做 1 sidebar）：

1. **craftDetail**：工艺/结构拆解（瓷砖如何烧、穹顶如何撑，300-500 字）
2. **controversy**：历史争议 + 双方证据（Dandolo 是否被毒杀）
3. **hiddenAgenda**：建造真实动机（6 宣礼塔是误听还是苏丹政治试探）
4. **architectFigure**：具名工匠/建筑师长小传（Sinan 来圣索菲亚学了什么）

## 反馈 2 · 参观时段最不 OTA 的形态

**叙事段落 > 矩阵**。

OTA 式 = 决策工具：季节×早晚矩阵 · ★评分 · 红黄绿 · "最佳时间"字样——🟡 "红色紧迫感三件套"近亲。

Cereal/LP/GAC 全走**编辑内容型**：
- Cereal Kyoto 四季光线：**诗意描述**（"11 月霜晨 6:30 的光切过清水寺的檐角"）
- LP "A Day in Istanbul"：**一天小短文**（6:00 宣礼声 / 13:00 瓷砖反光 / 17:00 金光）
- GAC seasonal：文学化专题集合页

**判断**：决策工具永远输给编辑内容。用户看到"最佳时间+表格"的一秒，产品从"文化向导"变"机票 app"。

**`visitTiming` 字段推荐**：

```js
// ❌ 矩阵（OTA 化注定）
visitTiming: { spring: {morning: "★★★", ... } }

// ✅ 叙事（精选 2-3 条编辑段）
visitTiming: [
  { when: "冬季日出前", body: "雪后第一缕光切过穹顶，西北角撒拉弗天使的金箔被照亮——一年只 3 天" },
  { when: "夏日黄昏", body: "6 根宣礼塔剪影压在博斯普鲁斯海面，鸥群和祷告声混在一起" }
]
```

**禁用字样**："最佳时间" / "推荐时段" / "★评分" / "高/中/低峰" / "仅限/必须"——任一 = OTA 化。
