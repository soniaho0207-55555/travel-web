# research: 反向案例 - OTA 廉价感（Booking / 携程 / 马蜂窝） - 2026-04-16

## 背景

前两次研究（Cereal、Linear）都是**正向 anchor**——「我希望 travel-web 长这样」。这次反过来：拆解「我**绝不希望** travel-web 长这样」的具体形态。

对象是 OTA（Online Travel Agency）家族：Booking.com / Expedia / TripAdvisor 国际版，以及中国侧的携程 / 马蜂窝 / 去哪儿。它们不是"做得差"，而是"被商业模式绑死，只能这么做"——红色紧迫感 banner、社会证明数字、徽章堆、广告插入——每一条都是为了多卖一晚酒店，每一条都和"杂志感"相反。

travel-web 不卖酒店，但**廉价感的视觉语法是会传染的**：哪怕只是城市内容站，一旦不小心抄了 OTA 的视觉模式（哪怕是出于"丰富信息"的好意），气质立刻就掉到马蜂窝那一档。

## 廉价感模式拆解（值得识别的反面）

### 1. 红色紧迫感 banner / 倒计时

**典型样式**：
- 红色背景 + 白字 + 粗体："Only 1 room left!" / "Deal ends in 02:35:11" / "仅剩 3 间"
- 位置：搜索结果卡片右上角徽章 + 详情页顶部横条
- 配套：倒计时数字每秒跳动，强化时间压力

**心理机制**：紧迫感（FOMO）+ 稀缺性偏差。Booking 在 Nunspeet 测试里被发现"84% 不可用"是假数据。

**廉价感判定**：**任何红色 + 倒计时 + 数量提示**三合一出现 = 立刻 OTA dark pattern。哪怕只出现一个红色"特惠""仅剩"，都已经踩进这个家族。

travel-web 不该有任何"仅剩 / 限时"——城市内容不存在售罄。

### 2. 徽章堆叠（Badge Soup）

**典型样式**：
- 一个景点 / 酒店卡片上贴 3-5 个不同颜色徽章
- "必去 TOP10" 红 + "热门" 橙 + "精选" 金 + "Travelers' Choice 2024" 绿 + "98% 推荐" 蓝
- 颜色彼此打架，没有视觉等级

**心理机制**：试图用多重"权威"叠加说服。结果是——读者根本不读，只感到"这页面很吵"。

**廉价感判定**：**一张卡片上出现 ≥3 个不同颜色徽章** = 廉价比拼广告路数。

正向反例：Cereal City Guide 一个景点就一段话 + 一张图，没有徽章。徽章本身就是"我替你判断"，但 Cereal 的"判断"在选择本身（出现在书里就是入选），不在贴标签。

### 3. 大图轮播 banner 占第一屏

**典型样式**：
- 1200×400 / 1920×600 大 banner
- 自动每 4-5 秒切换
- 内容是"暑期特惠 7 折起"+大字促销
- 上面还要叠 "立即抢购" 圆角按钮

**心理机制**：把"内容入口"让给"促销入口"。门户时代（2008-2014）标配。

**廉价感判定**：**第一屏主视觉是自动轮播 banner** 而不是内容首项 = 2010 年门户路数。

正向反例：Linear / Stripe / Apple 官网——首屏永远是一个静态、克制、有张力的画面，不是滚动促销。

### 4. 假社会证明数字

**典型样式**：
- "12,453 人想去 / 8,927 人收藏 / 327 人正在浏览"
- 红字、大号、置顶或卡片角
- 数字精确（不是约整）让你"觉得是真的"

**心理机制**：bandwagon effect（从众）+ FOMO 双倍打击。

**廉价感判定**：**任何精确大数字+红色+焦虑性动词（想去 / 浏览 / 抢）**= 假社会证明。

travel-web 这种内容站根本没有"X 人在看"这种数据，硬造就是廉价。如果真有数据，也该用克制的方式呈现（小灰字、整数、靠后），不是焦虑性突出。

### 5. 列表里突然冒出广告卡

**典型样式**：
- 城市卡片流里第 3 个突然是"携程酒店预订" / "签证服务" 卡
- 样式刻意接近内容卡（让你误点）但又略不同（图风更"促销"）
- 有时角上有小灰字"广告"或"推广"——那是合规标记，不是诚意

**心理机制**：广告 cosplay 内容。

**廉价感判定**：**列表中出现一个样式略不同、内容明显是商品的卡片**= 站点要赚钱的廉价信号。即使是合规广告也廉价——因为它在告诉用户"我们的内容流不值得保持纯度"。

travel-web 是 GitHub Pages 静态站，不会有广告——但要警惕未来万一加"推荐酒店"链接卡，是同一种污染。

## 共同特征（廉价感的视觉语法）

把上面 5 条抽象一层，廉价感共享几个信号：

1. **饱和度高**——红、橙、亮蓝、紫，几乎没有灰
2. **焦虑性动词**——抢 / 仅剩 / 即将 / 错过 / 想去 / 在看
3. **数字突出**——大字号、彩色、精确（非整数让人觉得"是真数据"）
4. **元素打架**——3+ 个东西同时在喊"看我"
5. **运动元素多**——轮播自动滑、倒计时跳秒、"X 人在看"实时刷新

反过来，"高级感"的视觉语法几乎是这五条的反面：低饱和、克制动词、数字小、视觉等级清楚、运动只服务功能。

## 不值得学 / 反而要避开的（这章特殊）

这次研究**整个就是"不值得学"**。但我要警惕一个陷阱：

**"反 OTA = 必走 Cereal"** 是错的。
中间还有一个被 OTA 廉价感污染了，但本质不该被一棍子打死的需求——**实用信息**。门票价、开放时间、推荐参观时长——这些不是 dark pattern，是用户真的需要的事实。

要避免的不是"展示信息"，而是**"用焦虑营销的样式包装信息"**。同样一个开放时间，可以写成：
- ❌ 廉价：「⏰ 仅今日 9:00-17:30 开放！⚡」红色背景
- ✅ 克制：「9:00–17:30（周一闭馆）」灰字小号

## 可迁移到 travel-web 的灵感（自查清单）

这次研究**不产生新做法**，产生**自查清单**。每次 UX 看页面问 5 个问题：

1. 有没有任何红色背景 banner / 倒计时？
2. 一张卡片上有没有 ≥3 个徽章？
3. 第一屏主视觉是不是自动轮播？
4. 有没有"X 人想去""X 人正在看"这种营销数字？
5. 列表里有没有混入"非内容"的卡片？

任何一条 yes —— 立刻像马蜂窝 / Booking 那一侧靠了一格，要回头看是不是出于"丰富感"的好意但走偏了。

---

## 一句话总结

**廉价感是有视觉语法的**：高饱和 + 焦虑动词 + 大数字 + 元素打架 + 自动运动。任何一条出现在 travel-web 上，都是被 OTA 商业模式污染的痕迹——哪怕动机是好的（信息丰富、突出重点），结果都会让页面气质从「Cereal 那一档」掉到「马蜂窝那一档」。
