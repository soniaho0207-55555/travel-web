# demand research: 社区 - B 站 历史 UP 主 - 2026-04-18

## 输入源

- **来源类型**：社区（B 站 / 哔哩哔哩历史区）
- **对象**：B 站历史 UP 主 + 相关视频元数据 + 二手用户评价
- **样本**：4 个 B 站视频元数据真抓取 + 4 次 WebSearch 综合 + 1 次腾讯新闻一手
- **抓取方式**：WebFetch 直接（B 站视频页 SSR 部分可拿到元数据）+ WebSearch 二手补充
- **关键局限说明（必读）**：
  - ✅ B 站视频页 WebFetch 可拿：标题/UP/简介/播放/赞/币/评论数/弹幕数/时长/发布日期
  - ❌ B 站视频页 WebFetch **拿不到**：评论区原文 + 弹幕原文（需要 AJAX 后加载）
  - ❌ B 站搜索/空间页 WebFetch 也拿不到（lazy load）
  - ❌ Chrome MCP 拒 `bilibili.com`（与小红书 / Reddit 同样的硬限）
  - **后果**：原研究目标"视频受众的文字需求信号源（评论区）"**降级**为"视频选题策略 + 数据规模信号 + 二手用户评价"

## 共性需求（≥2 个独立来源支撑）

### 1. 长篇剧情化历史叙事 = 中文头部模式（**与 Craig Mod / 马伯庸 形成中英三方验证**）

3+ 来源指向同一结论：B 站历史区头部 UP 主的成功公式 = **30-60 分钟长视频 + 剧情化叙事 + 人格化 IP**。

**核心证据**：
- 渤海小吏《安史之乱》3035 万播放 / 59 分钟 / 1.6 万评论 = 单期硬数据
- 渤海小吏粉丝 419 万 / 涨粉 2024 年最快 = 模式可复制成功
- 二手评价："用天津人说书的方式讲历史" = 剧情化 ≠ 学术化
- 与已有池条目 [Craig Mod editorial 段落长度 SOP 放开] 🟠 v2.8 强同方向 — 之前 Craig Mod 是 5 篇 ≥2000 字英文证据，B 站这条是 3000 万播放中文证据

**映射到我们 app**：
- 加固 [Craig Mod 段落长度] 条目证据权重，从单语言（英文）升级为**双语言双场景（英文长文 + 中文长视频）共识**
- PM 可直接使用：editorial 长段落 SOP 在 v2.8 应该实施而非"评估"
- 不重复立条，但建议 PM 把 Craig Mod 条目提档（🟠 → 🟢）

### 2. 历史 IP 的多媒介呼应 = 城市流量的复合驱动

3 源（渤海小吏《大唐气象》40+ 期 + 马伯庸《长安的荔枝》电影 + 之前 [马伯庸] 池条目）。

**核心信号**：用户对一个时代/一座城的兴趣不是单点产生的，是被**同期多媒介内容反复触达**形成的 mindshare。当渤海小吏在 B 站讲大唐 + 马伯庸在大屏放长安 + 抖音/小红书在传播唐宫舞 = 用户的"想去西安/洛阳"被反复激活。

**映射到我们 app**：
- 与已有池条目 [马伯庸 三明治理论 + 小人物切入] 强共鸣
- 新维度：**城市页可加"近期相关 IP / 视频 / 影视"模块**（不是推广，是"如果你刚看了 X，应该来这看 Y"的交叉触达）
- PM 可操作：可作为 [简单 funFact 字段] 的扩展，加 relatedMediaIPs 字段（电影/纪录片/UP主视频引用）

### 3. 唐宫夜宴现象 = 内容引爆地点的真实案例（"反向文旅"）

2 源确认（B 站页 metadata + 多个二手综合）。

**核心信号**：158 万 B 站播放 + 全网 20 亿播放的一段 5 分钟舞蹈，**直接把河南博物院从冷门博物馆变成网红打卡地**。这是中文互联网的标志性案例。

**映射到我们 app**：
- 这是**反向论据**支撑 [Craig Mod editorial 时代叠加] 🟢 v2.7 + [知乎 文学/名人关联] 🟢 v2.7 — 内容创造旅游需求是真实可量化的
- 我们 app 不直接做内容引爆（不是 UGC 不是视频平台），但可以**站在被引爆方的接收位**：当一个城市被某个文化事件引爆时，我们的城市页应该能立刻提供深度内容承接流量
- 不新增条目，加固现有内容深度方向

### 4. "剧情体" > "百科体" = 中文历史内容的核心分野

2 源确认（渤海小吏 vs 教师型 UP 主对比 + 二手评价用语）。

**核心信号**：B 站头部历史 UP 主全部走"叙事化 / 戏剧化 / 人格化 IP"路线，而不是"课本知识可视化 / 教师讲解" 路线。后者也存在（沙盘上的战争 / 正直讲史 / 历史陈老师）但属第二梯队。

**映射到我们 app**：
- 加固 [Craig Mod whyVisit 反主流叙事] + [马伯庸 三明治理论] + [Substack AI 免疫]
- 我们 app 的 editorial 应该明确选**剧情体路线**（有视角、有矛盾、有金句）而非百科体（只有事实+年代）
- 不新增条目，但是一个明确的**已有方向的强化证据**

## 单点线索（仅 1 个来源）

### 1. "嘴里塞着棉花跳舞" = 弹幕的"细节考据"特性

唐宫夜宴页 metadata 里看到 "唐宫少女都是嘴里塞着棉花在跳舞" 这条提示。这是 B 站弹幕的典型特征——观众沉迷于"破解细节"。

**含义**：中文用户对历史/文化内容**有强烈的"内幕考据"兴趣**。我们 app 的 funFact / didYouKnow 字段方向正确——找"非显而易见的细节"是真实需求。

### 2. "渤海郡王" / "通辽宇宙" = 历史 UP 主的人格化 IP

二手综合：渤海小吏粉丝拒绝叫"小吏"叫"郡王"；小约翰可汗的"通辽宇宙"已成亚文化标签。B 站历史 UP 主 = 强人格 IP，不是匿名内容生产者。

**含义**：我们 app 是匿名编辑产品，没有"作者人格"。这条信号目前我们不能直接吸收（个人 IP 难以快速建立），但是**未来如果做"专家/编辑专栏"应注意人格化**。

### 3. 评论率倒挂规律（小视频高评论率，大视频低评论率）

唐宫夜宴 158 万播放 / 10.7 万评论 = 6.8% 评论率
渤海小吏安史 3035 万播放 / 1.6 万评论 = 0.05% 评论率

**含义**：B 站评论率随播放量上升而下降。这反映**头部内容受众的"消费而非互动"特性**。对我们 app 的启发：不要假设受众会大量评论/留言，主战场是阅读体验本身。

### 4. 长视频时代的内容形态（30-60 分钟主流）

3 个渤海小吏视频时长全部 30-60 分钟，这与传统短视频时代（15 秒-3 分钟）形成对比。中文用户对长内容的耐受度被严重低估。

## 噪音 / 已排除

- **B 站会员体系 / 弹幕礼仪 / 充电系统**：B 站平台内部机制，与我们 app 无关。不进。
- **UP 主商业化（恰饭）模式**：业内信息，与用户需求无关。不进。
- **唐宫夜宴的"国潮"营销战略**：宏观文化现象，单独研究价值，但对我们当前 PRD 不直接驱动。不进。
- **马伯庸《长安的荔枝》电影评分**：商业 IP 评价，已在马伯庸研究覆盖。不进。
- **小约翰可汗的世界国别史选题（亚欧非拉小国）**：超出"历史名城"产品范围。不进。

### 幸存者偏差警告

B 站历史区头部 UP 主只代表"愿意看 30-60 分钟历史长视频的中文用户"，约几百万级活跃 UP 主主受众。这个样本：
- ✅ 与我们 History Reader persona 高度重合
- ⚠️ 不代表小红书 60 秒短视频用户（Armchair / Parent）
- ⚠️ 评论原话未直接获取，"评论共性"段全部来自 WebSearch 综合二手

## 纪律备注

- **B 站评论原文缺失**：本研究最大缺陷是评论区原文 0。如 PM/CEO 想要"用户原话"颗粒度的信号，需要：(a) CEO 手动复制 B 站评论 (b) PMO 配 Chrome MCP 白名单加 `bilibili.com`。本研究先用元数据 + 二手综合补救。
- **下次扫描窗口**：B 站作为视频平台，月度变化大。建议每月扫一次头部历史 UP 主新视频元数据。
- **领域局限**：B 站历史区主要是中国史，世界历史名城（罗马/伊斯坦布尔/京都）覆盖少，需要其他来源补。
