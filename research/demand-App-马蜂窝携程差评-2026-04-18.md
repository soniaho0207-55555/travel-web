# demand research: App - 马蜂窝 + 携程 差评 - 2026-04-18

## 输入源

- **来源类型**：App 评论 + 业内评测（中文 OTA / UGC 旅游平台）
- **对象**：马蜂窝（UGC 攻略社区起家）+ 携程（OTA 平台基因）
- **样本**：4 个一手页面 + 4 次 WebSearch 综合
- **抓取方式**：WebFetch（一手成功率 100%，所有目标页都拿到正文）

### 质量自检

| 指标 | 数量 |
|---|---|
| 一手源 | 4 个（界面新闻深度报道 / 新京报抄袭事件 / 36kr 500 用户调研 / App Store 评论页 + 差评原文 2 条） |
| 二手源 | 4 次 WebSearch 综合（人人都是产品经理 ×2 / 旅讯 / 36kr） |
| 抓取失败 | 0 |
| 证据强度 | **强**（≥3 一手 + 4 篇深度文 + App Store 用户原话） |

### 研究定位说明

本研究本质是**反面教材研究**（与 Culture Trip 同类）。马蜂窝走过了"UGC 内容平台 → 加 OTA → 被多平台合围 → 边缘化"的完整失败弧线，是中文 OTA + 内容产品的最完整教学案例。携程作为对比，是"OTA 基因 + 内容辅助"的成功案例。

### 携程作为对比（OTA 基因 vs UGC 基因）

| 维度 | 携程 | 马蜂窝 |
|---|---|---|
| 基因 | 交易平台原生 | UGC 内容社区 |
| 内容定位 | "美中不足"（人人评测原话）— 攻略是辅助 | 早期核心，后期被稀释 |
| 用户粘性来源 | 机票酒店预订习惯 | 攻略阅读 → 已被替代 |

**关键对比**：携程证明"OTA 主战场 + 内容辅助"路线可成功；马蜂窝证明"内容主战场 + OTA 后加"路线难成。**这告诉我们：内容平台 + 商业化是高度风险的组合**。

## 共性需求（≥2 个独立来源支撑）

### 1. **PC 长文形态在移动端死亡 + 但 B 站长视频活着 = 移动端长内容必须"视觉化分段"**（**新方向**）

3+ 源支撑（界面新闻 + 用户原话 + WebSearch 综合）。

**核心证据**：
- 界面新闻明确："万字蜂首到了移动端与九宫格图文 PK，就如同博客遇上微博，注定会逐渐没落"
- 用户原话："上千字游记找不到一个推荐景点，不适合当作工具类 APP 使用"
- 但同时 [B 站研究] 已验证：渤海小吏《安史之乱》59 分钟视频 3035 万播放——长内容在移动端不是死路

**关键区分**：长视频活 vs 长文死，差异在哪？答：**视觉锚点**。视频天然有时间轴 + 画面切换 + 弹幕互动 = 用户有"位置感"。长文如果纯文字海 = 用户失去位置感 → 弃读。

**映射到我们 app**：
- **新条目候选**：editorial 长内容必须配套"视觉锚点系统"——分段标题 / 内嵌图 / 时间线卡 / 地图标注 / 关键金句卡
- 与 [Craig Mod editorial 段落长度 SOP] 形成更精准的执行指引：不是简单"放开段落长度"，而是"放开 + 加视觉锚点"
- 与 B 站研究的"长内容容忍度铁证"互为正反——B 站验证可行性，马蜂窝验证执行细节

**这是 demand-pool 的新维度**——之前所有"内容深度"信号都没讨论"内容载体的视觉结构"，本研究补上。

### 2. UGC 抄袭/软文化 = 编辑产品的天然护城河（**反向加固已有方向**）

3 源（新京报抄袭事件 + WebSearch 软文综合 + 36kr 调研）。

**核心证据**：
- 85% 评论被指抄袭——UGC 平台的核心资产可信度危机
- "推荐页的内容太杂、太乱，软文和广告越来越多"
- 与 [CT 反面教材] 失败方式不同（CT 是雇人写 listicle，马蜂窝是 UGC 用户造假），但**结果相同：内容可信度归零**

**映射到我们 app**：
- 我们是**编辑型产品**（PM 写 PRD → Dev 实现），不开 UGC——这是结构性优势而非短期选择
- 加固 [Substack AI 免疫] + [CT 反面教材] 已有方向
- 不立新条，但建议 PM 在 PRD "产品原则"段明确写："**不开 UGC**"作为产品红线

### 3. 内容平台变现陷阱 = "内容主+ OTA 后加"路线难成（**强化已有反面教材**）

2 源（36kr 调研 + 界面新闻 + CT 已有数据）。

**核心证据**：
- 马蜂窝用户调研："机票酒店？没想过在上面订，习惯携程了"
- "一个目的地里只有几篇精选攻略，可变现转化的商品依然很少"
- 携程对比：交易平台原生 + 内容辅助 = 成功
- 与 [CT 反面教材] 重叠（commerce 污染 editorial）

**映射到我们 app**：
- **战略含义**：我们 app 如果未来商业化，**最危险的路径就是"内容深 + 自建 OTA"**——这是马蜂窝路径
- **可行的路径**：纯订阅（Craig Mod / Substack）/ 联盟跳转极简（不抢 GMV，只赚 CPS）/ B2B 内容授权
- 不立新条，加固 [CT 反面教材] + [GAC 产品象限定位]

### 4. 移动端 95 后断层 = 内容形态代际差异

1 源但极重要（界面新闻明确指出"95 后网友普遍未使用过马蜂窝"）。

**核心证据**：
- 95 后没有"PC 长文蜂首"的记忆，直接成长在小红书短图文 + 抖音短视频 + B 站长视频环境
- 他们对"万字游记"无感，对"60 秒探店视频" 有感
- 但同时 95 后又会看 60 分钟历史长视频（B 站验证）

**映射到我们 app**：
- 我们的目标用户 (History Reader 老周 41 岁 + Planner 林姐 32 岁) **不是 95 后核心**
- 但下一代 (95 后到 00 后) 的"内容耐受样式"已经清晰：**短图文+短视频+长视频** 三段式，长文独占领死域
- 我们 app 是文字产品（长文向）→ **必须靠"视觉锚点"才能让长文获得长视频的"位置感"**
- 与上述 #1 互验

## 单点线索（仅 1 个来源）

### 1. App Store 客服/退款问题（不是内容问题，是商业化问题）

App Store 差评显示：客服打 17 个电话无回复 / 退款 14 个工作日。这是马蜂窝**商业化做不好的具体表现**——不是内容问题。

**含义**：商业化能力不是简单"开个 OTA"就行，需要客服/退款/支付/物流全链路。这进一步验证我们不应轻易做 OTA。

### 2. 民宿信息不足（垂类痛点）

36kr 调研第二痛点："民宿信息不足（特色民宿需要详细点评但展示不够）"。

**含义**：用户期待的"内容深度"是**带细节的人物/场景描述**，不是纯标签。这与 [Craig Mod 具名店主] + [马伯庸 小人物切入] 同方向，但属用户原话佐证。

### 3. 算法推荐缺失

36kr 调研第三痛点："缺乏千人千面推荐算法，导致用户转向小红书"。

**含义**：不是核心信号——我们 app 当前是静态站，无算法。但**长期**如果做推荐，必须做"千人千面"否则会被小红书替代型平台碾压。**远期信号，非当下需求**。

### 4. 美团/抖音/小红书的合围

WebSearch 综合：马蜂窝衰退的外部原因之一是被多个平台从不同方向挤压。

**含义**：垂类深度是护城河——我们的"历史名城"垂直是明确的，比马蜂窝当年"全品类旅游"窄得多，反而抗合围能力强。**加固定位选择**。

## 与已完成研究的最终交叉总结（8 个研究全闭环）

| 维度 | 正面阵地 | 反面阵地 |
|---|---|---|
| **内容深度** | Craig Mod / 马伯庸 / Substack / B 站 | LP（深度浅） / 马蜂窝（PC 长文死） / CT（SEO 堆砌） |
| **本地视角** | Craig Mod 具名店主 / 马伯庸 小人物 / Substack 本地作者 | CT 配额制撰稿人远程谷歌搬运 |
| **载体形式** | B 站长视频自带视觉锚点 / Craig Mod 长文配图 | 马蜂窝纯文字海 |
| **变现路径** | Craig Mod 纯订阅 / Substack 付费墙 / GAC 公益 | 马蜂窝 内容→OTA 失败 / CT 内容→OTA→被收购 |
| **垂类策略** | 我们 app 历史名城 = 明确垂直 | 马蜂窝 全品类 = 被合围 |
| **UGC 策略** | 我们 = 编辑产品 = 不开 UGC | 马蜂窝 = UGC 造假 / AO = UGC 但严控（贡献者榜） |

**8 个研究的最终结论**：我们 app 处在一个**反 Culture Trip + 反马蜂窝 + 学 Craig Mod + 学 B 站长内容**的明确产品位置。这是 demand-pool 累积到现在的最硬战略发现。
