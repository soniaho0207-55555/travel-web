## Dev 落地说明

1. **加粗首句**：渲染 whyVisit 时，取第一个 `。` 或 `\n\n` 之前的文本，用 `<strong>` 或 `font-weight: 700` 渲染
2. **{tag} 行**：已有 `{tag: ...}` 的景点，tag 行本身是独立 UI 组件（J-02 spec），加粗首句从 tag 行下一段开始
3. **weak 条目**：PM 在 v2.8 Dev 前完成 13 条 whyVisit 首句重写，直接改 data.js 对应字段——Dev 不需要做判断，只需无脑加粗第一句
4. **不改结构**：whyVisit 字段的后续段落不做任何修改，只改首句
